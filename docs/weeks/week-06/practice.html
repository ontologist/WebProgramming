About This Practice / この練習について

This is a guided in-class practice exercise. We will work through it together during class. You will take your moving ball from Session 5 and add wall bouncing, one wall at a time.
これは授業内のガイド付き練習です。授業中に一緒に進めていきます。セッション5で作った動くボールに、壁の跳ね返りを1つずつ追加していきます。

This exercise is not graded. Its purpose is to give you hands-on experience with if statements and boundary detection. You are building on your game -- the same code you will continue to develop in future sessions.
この練習は採点されません。if文と境界検出の実践経験を積むことが目的です。ゲームの上に構築しています -- 今後のセッションでも引き続き開発する同じコードです。

What you need: Your game file from Session 5 (the moving ball animation) open in VS Code. If you do not have it, use the starter code in Step 1 below.

必要なもの： セッション5のゲームファイル（動くボールのアニメーション）をVS Codeで開いてください。持っていない場合は、下のステップ1のスターターコードを使ってください。

Step 1: Start with Your Moving Ball / 動くボールから始める

Open your game file from Session 5. Your code should look like this. If you do not have it, create a new file called game.html and type this code:
セッション5のゲームファイルを開いてください。コードはこのようになっているはずです。持っていない場合は、game.htmlという新しいファイルを作成し、このコードを入力してください：

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>My Breakout Game</title>
</head>
<body>
    <canvas id="myCanvas" width="480" height="320"
        style="border: 1px solid #000;"></canvas>
    <script>
        var canvas = document.getElementById("myCanvas");
        var ctx = canvas.getContext("2d");
        var x = canvas.width / 2;
        var y = canvas.height - 30;
        var dx = 2;
        var dy = -2;
        var ballRadius = 10;

        function draw() {
            ctx.clearRect(0, 0, canvas.width, canvas.height);
            ctx.beginPath();
            ctx.arc(x, y, ballRadius, 0, Math.PI * 2);
            ctx.fillStyle = "#0095DD";
            ctx.fill();
            ctx.closePath();
            x += dx;
            y += dy;
        }

        setInterval(draw, 10);
    </script>
</body>
</html>

Test it first

Open this file in your browser. You should see a blue ball that moves diagonally and then flies off the screen. That is the problem we are going to fix!
このファイルをブラウザで開いてください。斜めに動いて画面外に飛んでいく青いボールが見えるはずです。それが今から修正する問題です！

Step 2: Add Right Wall Detection / 右壁の検出を追加

Let's start by making the ball bounce off the right wall. We need to add an if statement that checks if the ball is about to go past the right edge.
まず、ボールが右壁で跳ね返るようにしましょう。ボールが右端を超えようとしているかチェックするif文を追加する必要があります。

In your draw() function, add this code before the lines x += dx; and y += dy;:
draw()関数の中で、x += dx; と y += dy; の行の前に、このコードを追加してください：

// Bounce off the right wall
if (x + dx > canvas.width) {
    dx = -dx;
}

What this does:

x + dx -- calculates where the ball will be next frame.
> canvas.width -- checks if that position is past the right edge.
dx = -dx -- reverses the horizontal direction (e.g., 2 becomes -2).
x + dx -- 次のフレームでのボールの位置を計算する。> canvas.width -- その位置が右端を超えているかチェック。dx = -dx -- 水平方向を反転する（例：2が-2になる）。

Test it! Save and refresh your browser. The ball should now bounce off the right wall, but it still flies off the left, top, and bottom. We will fix those next.

テストしよう！ 保存してブラウザを更新してください。ボールが右壁で跳ね返るはずですが、左、上、下ではまだ飛んでいきます。次にそれらを修正します。

Step 3: Add Left Wall Detection / 左壁の検出を追加

Now let's handle the left wall. Add this code right after the right wall check:
次に左壁を処理しましょう。右壁のチェックの直後に、このコードを追加してください：

// Bounce off the left wall
if (x + dx < 0) {
    dx = -dx;
}

Or combine both into one statement:

Since both the left and right wall use dx = -dx, you can write them as one if statement using the OR operator (||):
左壁と右壁の両方がdx = -dxを使うので、OR演算子（||）を使って1つのif文にまとめることもできます：

// Bounce off left OR right wall
if (x + dx > canvas.width || x + dx < 0) {
    dx = -dx;
}

Test it! The ball should now bounce off both left and right walls. It bounces horizontally but still escapes through the top and bottom.

テストしよう！ ボールが左右の壁で跳ね返るはずです。水平方向には跳ね返りますが、上下にはまだ逃げます。

Step 4: Add Top Wall Detection / 上壁の検出を追加

The top wall works the same way, but for the y-axis. The top of the canvas is y = 0. Add this code after your horizontal wall check:
上壁も同じように機能しますが、y軸に対してです。キャンバスの上端はy = 0です。水平方向の壁チェックの後に、このコードを追加してください：

// Bounce off the top wall
if (y + dy < 0) {
    dy = -dy;
}

What this does:

y + dy -- calculates the ball's next vertical position.
< 0 -- checks if it has gone above the top edge.
dy = -dy -- reverses the vertical direction.
y + dy -- ボールの次の垂直位置を計算。< 0 -- 上端を超えたかチェック。dy = -dy -- 垂直方向を反転。

Step 5: Add Bottom Wall Detection / 下壁の検出を追加

Add the bottom wall detection. You can combine it with the top wall check:
下壁の検出を追加してください。上壁のチェックと組み合わせることができます：

// Bounce off top OR bottom wall
if (y + dy > canvas.height || y + dy < 0) {
    dy = -dy;
}

Test it! Your ball should now bounce off ALL four walls! It will keep bouncing forever, never escaping the canvas. Congratulations -- you just implemented collision detection!

テストしよう！ ボールが4つの壁すべてで跳ね返るはずです！永遠に跳ね返り続け、キャンバスから逃げません。おめでとうございます -- 衝突検出を実装しました！

Your draw() function should now look like this:

Here is the complete function with all boundary checks added:
すべての境界チェックを追加した完全な関数がこちらです：

function draw() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    ctx.beginPath();
    ctx.arc(x, y, ballRadius, 0, Math.PI * 2);
    ctx.fillStyle = "#0095DD";
    ctx.fill();
    ctx.closePath();

    // Bounce off left and right walls
    if (x + dx > canvas.width || x + dx < 0) {
        dx = -dx;
    }
    // Bounce off top and bottom walls
    if (y + dy > canvas.height || y + dy < 0) {
        dy = -dy;
    }

    x += dx;
    y += dy;
}

Step 6: Fix the Edge Clipping / 端の食い込みを修正

Look closely at your bouncing ball. Do you notice that it seems to partially disappear into the wall before bouncing back? This is because we are checking the ball's center point, not its edge.
跳ね返るボールをよく見てください。跳ね返る前に壁に部分的に消えているように見えませんか？これはボールの端ではなく中心点をチェックしているためです。

Fix this by accounting for ballRadius in all four boundary checks. Replace your existing boundary code with:
4つの境界チェックすべてでballRadiusを考慮して修正してください。既存の境界コードを以下に置き換えます：

// Bounce off left and right walls (accounting for radius)
if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) {
    dx = -dx;
}
// Bounce off top and bottom walls (accounting for radius)
if (y + dy > canvas.height - ballRadius || y + dy < ballRadius) {
    dy = -dy;
}

Why this works:

Right wall: Instead of checking if center > canvas.width, we check if center > canvas.width - ballRadius. This means the ball's right edge (center + radius) will just touch the wall.
右壁： 中心 > canvas.widthではなく、中心 > canvas.width - ballRadiusをチェックします。これにより、ボールの右端（中心 + 半径）がちょうど壁に触れます。

Left wall: Instead of checking if center < 0, we check if center < ballRadius. This ensures the ball's left edge (center - radius) touches x = 0.
左壁： 中心 < 0ではなく、中心 < ballRadiusをチェックします。これにより、ボールの左端（中心 - 半径）がx = 0に触れます。

Test it! The ball should now bounce cleanly off all walls without clipping. Its edge touches the wall exactly at the moment it reverses direction.

テストしよう！ ボールが食い込みなく全ての壁できれいに跳ね返るはずです。方向が反転する瞬間に、端がちょうど壁に触れます。

Step 7: Experiment! / 実験しよう！

Your bouncing ball is working! Now let's experiment with different values to build your intuition about how the variables affect behavior.
跳ね返るボールが動いています！変数が動作にどう影響するか直感を養うために、いろいろな値を試してみましょう。

Try these experiments:

1. Change the speed: Try setting dx = 4 and dy = -4. What happens? Try dx = 1 and dy = -1.
1. 速度を変更： dx = 4とdy = -4にしてみてください。何が起こりますか？dx = 1とdy = -1も試しましょう。

2. Change the angle: Try dx = 3 and dy = -1. The ball moves mostly horizontal! Try dx = 1 and dy = -3 for mostly vertical.
2. 角度を変更： dx = 3とdy = -1を試してください。ボールはほぼ水平に動きます！dx = 1とdy = -3でほぼ垂直にも。

3. Change the starting position: Try x = 50 and y = 50 to start from the top-left corner.
3. 開始位置を変更： x = 50とy = 50で左上隅から開始してみてください。

4. Change the ball size: Try ballRadius = 20 for a bigger ball, or ballRadius = 5 for a tiny one.
4. ボールのサイズを変更： ballRadius = 20で大きなボール、ballRadius = 5で小さなボールを試しましょう。

5. Change the color: Try ctx.fillStyle = "#FF0000" (red) or "#00FF00" (green).
5. 色を変更： ctx.fillStyle = "#FF0000"（赤）や"#00FF00"（緑）を試しましょう。

What happens if ballRadius is bigger than the canvas? Try ballRadius = 200 -- the ball gets stuck! This is because it is always past both walls simultaneously. This is a good lesson about edge cases in programming.

ballRadiusがキャンバスより大きいとどうなるか？ ballRadius = 200を試してみてください -- ボールが動けなくなります！常に両方の壁を同時に超えているためです。プログラミングにおけるエッジケースについての良い教訓です。
