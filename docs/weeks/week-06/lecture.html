2 Review of Session 5 / セッション5の復習

In Session 5, you learned how to animate a ball moving across the canvas. Here is the code we ended with:

セッション5では、キャンバス上でボールを動かすアニメーションの方法を学びました。前回のコードがこちらです：

var canvas = document.getElementById("myCanvas");
var ctx = canvas.getContext("2d");
var x = canvas.width / 2;
var y = canvas.height - 30;
var dx = 2;
var dy = -2;
var ballRadius = 10;

function draw() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    ctx.beginPath();
    ctx.arc(x, y, ballRadius, 0, Math.PI * 2);
    ctx.fillStyle = "#0095DD";
    ctx.fill();
    ctx.closePath();
    x += dx;
    y += dy;
}

setInterval(draw, 10);

What each variable means / 各変数の意味

Variable Purpose / 目的 Initial Value
x Ball's horizontal position / ボールの水平位置 canvas.width / 2 (center)
y Ball's vertical position / ボールの垂直位置 canvas.height - 30 (near bottom)
dx Horizontal speed and direction / 水平方向の速度と方向 2 (moving right)
dy Vertical speed and direction / 垂直方向の速度と方向 -2 (moving up)
ballRadius Size of the ball / ボールのサイズ 10 pixels

The animation loop: Every 10 milliseconds, the draw() function clears the canvas, draws the ball at its current position, then updates the position by adding dx and dy. This creates the illusion of smooth movement.

アニメーションループ： 10ミリ秒ごとに、draw()関数がキャンバスをクリアし、現在の位置にボールを描画し、dxとdyを足して位置を更新します。これにより滑らかな動きの錯覚が生まれます。

3 The Problem: Ball Flies Off Screen / 問題：ボールが画面外へ

If you run the code from Session 5, you will notice a problem: the ball moves across the canvas and then disappears off the edge. It never comes back. Once x exceeds canvas.width or y goes below 0, the ball is being drawn outside the visible area.

セッション5のコードを実行すると、問題に気づくでしょう：ボールがキャンバスを横切って端から消えてしまいます。二度と戻ってきません。xがcanvas.widthを超えたり、yが0以下になると、ボールは可視領域の外に描画されています。

What we have now / 今の状態

  • The ball moves diagonally across the screen
  • It exits through the top-right corner (because dx is positive and dy is negative)
  • It never stops or changes direction

What we want / 目指す状態

  • When the ball hits a wall, it should bounce off it
  • The ball should stay visible inside the canvas forever
  • It should bounce off all four walls: left, right, top, and bottom

Analogy / たとえ話: Imagine throwing a rubber ball inside a closed room. When it hits a wall, it bounces off and goes the other way. It does not pass through the wall. That is exactly the behavior we want to program.

閉じた部屋の中でゴムボールを投げることを想像してください。壁にぶつかると跳ね返って反対方向に進みます。壁を通り抜けることはありません。それがまさに私たちがプログラムしたい動作です。

To make the ball bounce, we need our code to detect when the ball reaches a wall and then react by reversing its direction. This requires two new concepts: if statements (to detect) and velocity reversal (to react).

ボールを跳ね返らせるには、ボールが壁に到達したことを検出し、方向を反転させて反応する必要があります。これには2つの新しい概念が必要です：if文（検出用）と速度の反転（反応用）。

4 Making Decisions: The if Statement / 判断を下す：if文

Until now, our code has been sequential -- it runs every line from top to bottom, no exceptions. But real programs need to make decisions. Should the ball bounce? Should the game end? Should we add a point to the score? These all require conditional logic.

これまで、コードは順次的でした -- すべての行を上から下へ例外なく実行していました。しかし実際のプログラムは判断を下す必要があります。ボールは跳ね返るべきか？ゲームは終了すべきか？スコアに点を追加すべきか？これらすべてに条件ロジックが必要です。

The if statement syntax / if文の構文

if (condition) {
    // This code runs ONLY when condition is true
    // 条件がtrueのときだけ、このコードが実行される
}

The structure has three parts:

  1. if -- the keyword that tells JavaScript "I want to make a decision" / JavaScriptに「判断したい」と伝えるキーワード
  2. (condition) -- the question to ask, inside parentheses. Must be true or false / 括弧の中の質問。trueまたはfalseでなければならない
  3. { code } -- the code to run if the condition is true, inside curly braces / 条件がtrueのとき実行するコード、波括弧の中

Real-life analogy / 実生活のたとえ:

  • "If it is raining, bring an umbrella." -- You only bring the umbrella when the condition (raining) is true. 「もし雨が降っていたら、傘を持っていく。」
  • "If the traffic light is red, stop the car." -- You only stop when the condition (red light) is true. 「もし信号が赤なら、車を止める。」
  • "If the ball hits a wall, reverse direction." -- We only reverse when the condition (hitting wall) is true. 「もしボールが壁にぶつかったら、方向を反転する。」

A simple example / 簡単な例

var temperature = 35;

if (temperature > 30) {
    console.log("It is hot today!");
}
// Output: "It is hot today!" (because 35 > 30 is true)
var temperature = 20;

if (temperature > 30) {
    console.log("It is hot today!");
}
// No output (because 20 > 30 is false, code inside {} is skipped)

Common mistake / よくある間違い: Do not forget the parentheses around the condition or the curly braces around the code. Both are required!

条件を囲む括弧とコードを囲む波括弧を忘れないでください。両方とも必須です！

5 Comparison Operators / 比較演算子

To write conditions, we need comparison operators. These compare two values and produce a boolean result: either true or false.

条件を書くには、比較演算子が必要です。これらは2つの値を比較し、ブール結果（trueまたはfalse）を生成します。

Operator Meaning / 意味 Example Result
> Greater than / より大きい 7 > 5 true
< Less than / より小さい 3 < 2 false
>= Greater than or equal to / 以上 5 >= 5 true
<= Less than or equal to / 以下 10 <= 8 false
=== Strictly equal to / 厳密に等しい 4 === 4 true
!== Not equal to / 等しくない 4 !== 5 true

Using comparison operators with variables / 変数との使用

var x = 150;
var canvasWidth = 480;

x > canvasWidth      // false (150 is not greater than 480)
x < canvasWidth      // true  (150 is less than 480)
x === 150            // true  (x is exactly 150)

// For our game:
x + dx > canvasWidth // Is the ball past the right wall?

Tip / ヒント: Use === (three equals signs) instead of == (two equals signs) for equality comparisons in JavaScript. The triple equals checks both value and type, which prevents subtle bugs.

JavaScriptでの等価比較には==（2つの等号）ではなく===（3つの等号）を使いましょう。トリプルイコールは値と型の両方をチェックするため、微妙なバグを防ぎます。

6 if/else and if/else if/else / if/else と if/else if/else

Sometimes you want to do one thing if a condition is true, and a different thing if it is false. That is what else is for.

条件がtrueのときに一つのことをし、falseのときに別のことをしたい場合があります。それがelseの役割です。

if/else / if/else文

if (condition) {
    // runs when condition is TRUE
} else {
    // runs when condition is FALSE
}

// Example:
var score = 3;
if (score >= 5) {
    console.log("You win!");
} else {
    console.log("Keep trying!");
}
// Output: "Keep trying!" (because 3 >= 5 is false)

if/else if/else (multiple conditions) / 複数の条件

var temperature = 25;

if (temperature > 35) {
    console.log("Very hot!");
} else if (temperature > 25) {
    console.log("Warm.");
} else if (temperature > 15) {
    console.log("Cool.");
} else {
    console.log("Cold!");
}
// Output: "Cool." (because 25 > 15 is true, checked after 25 > 35 and 25 > 25 failed)

For our bouncing ball: We primarily use simple if statements (without else), because we just need to check "is the ball at a wall?" and reverse direction if true. We do not need an else because if the ball is NOT at a wall, we simply do nothing special.

跳ね返るボールの場合： 主にシンプルなif文（elseなし）を使います。「ボールは壁にいるか？」をチェックし、trueなら方向を反転するだけです。壁にいない場合は何もしないので、elseは不要です。

7 Boundary Detection: Right and Left Walls / 境界検出：右壁と左壁

Now let's apply what we learned about if statements and comparison operators to detect when our ball hits the walls.

では、if文と比較演算子について学んだことを使って、ボールが壁にぶつかるタイミングを検出しましょう。

The canvas coordinate system / キャンバスの座標系

Canvas Boundaries / キャンバスの境界
  x = 0                              x = canvas.width
    |                                      |
    v                                      v
    +--------------------------------------+  <-- y = 0
    |                                      |
    |          Canvas (480 x 320)          |
    |                                      |
    |              o  <-- ball at (x, y)   |
    |                                      |
    |                                      |
    +--------------------------------------+  <-- y = canvas.height

Detecting the right wall / 右壁の検出

The ball hits the right wall when its x position exceeds the canvas width. We check where the ball will be after the next frame (x + dx), not where it is now:

ボールのx位置がキャンバスの幅を超えると、右壁にぶつかります。ボールが今どこにいるかではなく、次のフレーム後にどこにいるか（x + dx）をチェックします：

// Detect right wall
if (x + dx > canvas.width) {
    dx = -dx;  // reverse horizontal direction
}

Detecting the left wall / 左壁の検出

The ball hits the left wall when its x position goes below 0:

ボールのx位置が0を下回ると、左壁にぶつかります：

// Detect left wall
if (x + dx < 0) {
    dx = -dx;  // reverse horizontal direction
}

Combining both into one statement / 両方を1つの文にまとめる

Since both conditions trigger the same action (dx = -dx), we can combine them with the OR operator (||):

両方の条件が同じアクション（dx = -dx）を引き起こすので、OR演算子（||）で組み合わせることができます：

// Detect left OR right wall
if (x + dx > canvas.width || x + dx < 0) {
    dx = -dx;
}

Why check x + dx instead of just x? / なぜxではなくx + dxをチェックするのか？ We want to detect the collision before it happens, so we can reverse direction in time. If we waited until x was already past the wall, the ball would briefly appear outside the canvas.

衝突が起きる前に検出したいからです。そうすれば時間内に方向を反転できます。xがすでに壁を超えてから検出すると、ボールがキャンバスの外に一瞬現れてしまいます。

8 Boundary Detection: Top and Bottom Walls / 境界検出：上壁と下壁

The same logic applies to the vertical axis (y). The only difference is we check y + dy and reverse dy instead of dx.

同じロジックが垂直軸（y）にも適用されます。唯一の違いは、y + dyをチェックし、dxではなくdyを反転することです。

Detecting the top wall / 上壁の検出

// Ball hits top wall when y goes below 0
if (y + dy < 0) {
    dy = -dy;  // reverse vertical direction
}

Detecting the bottom wall / 下壁の検出

// Ball hits bottom wall when y exceeds canvas height
if (y + dy > canvas.height) {
    dy = -dy;  // reverse vertical direction
}

Combined vertical check / 垂直方向のチェックをまとめる

// Detect top OR bottom wall
if (y + dy > canvas.height || y + dy < 0) {
    dy = -dy;
}

All four walls together / 4つの壁をすべてまとめる

// Horizontal walls (left and right)
if (x + dx > canvas.width || x + dx < 0) {
    dx = -dx;
}

// Vertical walls (top and bottom)
if (y + dy > canvas.height || y + dy < 0) {
    dy = -dy;
}

Important pattern / 重要なパターン: Notice the symmetry. For the x-axis, we check x boundaries and reverse dx. For the y-axis, we check y boundaries and reverse dy. Each axis is handled independently.

対称性に注目してください。x軸ではxの境界をチェックしてdxを反転。y軸ではyの境界をチェックしてdyを反転。各軸は独立して処理されます。

9 Velocity Reversal: The Bounce / 速度の反転：跳ね返り

The key to making the ball bounce is one simple operation: dx = -dx (or dy = -dy). Let's understand exactly why this works.

ボールを跳ね返らせる鍵は、一つのシンプルな操作です：dx = -dx（またはdy = -dy）。なぜこれが機能するのか正確に理解しましょう。

What does negation do? / 否定は何をするのか？

// dx = 2 means: move 2 pixels to the RIGHT each frame
dx = -dx;
// Now dx = -2 means: move 2 pixels to the LEFT each frame

// dx = -2 means: move 2 pixels to the LEFT each frame
dx = -dx;
// Now dx = 2 means: move 2 pixels to the RIGHT each frame

Negation flips the sign of the number. A positive number becomes negative, and vice versa. The magnitude (speed) stays the same -- only the direction changes.

否定は数値の符号を反転します。正の数は負になり、その逆も同様です。大きさ（速さ）は同じまま -- 方向だけが変わります。

Velocity Reversal Visualized / 速度反転の可視化
Before hitting right wall:    After bounce:
dx = 2 (moving right -->)     dx = -2 (moving <-- left)

    o -->  |                      |  <-- o
   ball   wall                  wall   ball


Before hitting top wall:      After bounce:
dy = -2 (moving up ^)         dy = 2 (moving down v)

    ------                      ------
      ^                           |
      |                           v
      o                           o

Analogy / たとえ話: Think of a rubber ball hitting a wall. The ball does not slow down or speed up (in our simplified physics) -- it simply reverses direction. If it was going right at speed 2, after the bounce it goes left at speed 2. The negation operator (-) is our code's way of expressing this reversal.

壁にぶつかるゴムボールを考えてください。（私たちの単純化した物理学では）ボールは減速も加速もしません -- ただ方向が反転するだけです。速度2で右に進んでいたら、跳ね返り後は速度2で左に進みます。否定演算子（-）は、この反転をコードで表現する方法です。

10 Accounting for Ball Radius / ボールの半径を考慮する

If you test the code from the previous sections, you will notice a visual problem: the ball seems to clip into the wall before bouncing back. Part of the ball disappears behind the edge before it reverses.

前のセクションのコードをテストすると、視覚的な問題に気づくでしょう：ボールが跳ね返る前に壁に食い込んでいるように見えます。反転する前にボールの一部が端の後ろに消えます。

Why does this happen? / なぜこれが起こるのか？

The variable x represents the center of the ball, not its edge. When we check x + dx > canvas.width, we are checking if the center has passed the wall. But the ball's visible edge is at x + ballRadius, which reaches the wall before the center does!

変数xはボールの中心を表し、端ではありません。x + dx > canvas.widthをチェックするとき、中心が壁を通過したかを確認しています。しかしボールの目に見える端はx + ballRadiusにあり、中心より先に壁に到達します！

Ball Center vs. Ball Edge / ボールの中心 vs. ボールの端
Without radius correction:         With radius correction:
(ball clips into wall)             (ball bounces at edge)

         |                                   |
    [ o ]|  <-- center at wall,         [ o ]|  <-- edge touches wall,
         |      half the ball is             |      ball bounces here
         |      hidden behind wall           |

  x = canvas.width (too late!)      x = canvas.width - ballRadius (just right!)

The fix: subtract and add ballRadius / 修正：ballRadiusを引いたり足したりする

// WITHOUT radius correction (ball clips into walls):
if (x + dx > canvas.width || x + dx < 0) {
    dx = -dx;
}

// WITH radius correction (ball bounces at edge):
if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) {
    dx = -dx;
}
if (y + dy > canvas.height - ballRadius || y + dy < ballRadius) {
    dy = -dy;
}

Understanding each boundary / 各境界の理解

Wall / 壁 Without Radius With Radius Why / 理由
Right x + dx > canvas.width x + dx > canvas.width - ballRadius Ball edge is at x + ballRadius
Left x + dx < 0 x + dx < ballRadius Ball edge is at x - ballRadius
Top y + dy < 0 y + dy < ballRadius Ball edge is at y - ballRadius
Bottom y + dy > canvas.height y + dy > canvas.height - ballRadius Ball edge is at y + ballRadius

Remember / 覚えておこう: Always account for the ball's radius when doing boundary detection. Without it, the ball will visually clip through walls. This is a common source of bugs in game development.

境界検出のときは常にボールの半径を考慮してください。これがないと、ボールが視覚的に壁を突き抜けます。これはゲーム開発でよくあるバグの原因です。

11 Logical Operators: && (AND) and || (OR) / 論理演算子：&&（AND）と ||（OR）

We have already used || (OR) in our wall detection code. Let's formally learn both logical operators.

壁検出コードで既に||（OR）を使いました。両方の論理演算子を正式に学びましょう。

|| (OR) -- at least one must be true / ||（OR）-- 少なくとも1つがtrue

// || means "or" -- the whole expression is true if EITHER side is true

true || true    // true
true || false   // true
false || true   // true
false || false  // false (neither is true)

// Our game example:
if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) {
    // This runs if the ball hits the right wall OR the left wall
    dx = -dx;
}

&& (AND) -- both must be true / &&（AND）-- 両方がtrue

// && means "and" -- the whole expression is true only if BOTH sides are true

true && true    // true
true && false   // false
false && true   // false
false && false  // false

// Example: check if something is within a range
var x = 100;
if (x > 50 && x < 200) {
    console.log("x is between 50 and 200");
}
// true, because both (100 > 50) AND (100 < 200) are true

Analogy / たとえ話:

  • || (OR) is like a door with two keys -- either key can open it. You need at least one. 2つの鍵があるドア -- どちらか1つで開きます。
  • && (AND) is like a safe that needs two keys turned at the same time -- both are required. 2つの鍵を同時に回す必要がある金庫 -- 両方必要です。

When to use which? / どちらを使うか？

|| -- When you want to do something if any one of several conditions is met. "Bounce if ball hits right wall OR left wall."

&& -- When you want to do something only if all conditions are met simultaneously. "The ball is in the safe zone if x > 0 AND x < canvas.width."

12 Compound Conditions / 複合条件

A compound condition is a condition made by combining multiple simpler conditions with logical operators. We have been using these already!

複合条件は、複数のより単純な条件を論理演算子で組み合わせて作られる条件です。すでに使っていました！

Our wall detection uses compound conditions / 壁検出は複合条件を使っている

// This is a compound condition using OR:
if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) {
//  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^    ^^^^^^^^^^^^^^^^^^^^^
//  Condition 1: right wall                 Condition 2: left wall
//  (joined by || which means "or")
    dx = -dx;
}

You can combine as many conditions as you need / 必要なだけ条件を組み合わせられる

// Check if a point is inside a rectangle (using AND):
if (x > rectLeft && x < rectRight && y > rectTop && y < rectBottom) {
    console.log("Point is inside the rectangle!");
}

// Check multiple failure conditions (using OR):
if (lives === 0 || time === 0 || health <= 0) {
    console.log("Game Over!");
}

For today's code / 今日のコード: We primarily use || (OR) to check if the ball is at either of two opposing walls. In later sessions, we will use && (AND) for more complex collision detection (like checking if the ball hits the paddle).

今日は主に||（OR）を使って、ボールが2つの対向する壁のどちらかにいるかをチェックします。後のセッションでは、より複雑な衝突検出（ボールがパドルに当たったかなど）に&&（AND）を使います。

13 The Complete Bouncing Ball / 完全な跳ね返るボール

Here is the complete, working code for a ball that bounces off all four walls. This is the full HTML file you can create and run:

4つの壁すべてで跳ね返るボールの完全な動作コードがこちらです。作成して実行できる完全なHTMLファイルです：

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Bouncing Ball</title>
</head>
<body>
    <canvas id="myCanvas" width="480" height="320"></canvas>
    <script>
        var canvas = document.getElementById("myCanvas");
        var ctx = canvas.getContext("2d");

        // Ball position (starts near center-bottom)
        var x = canvas.width / 2;
        var y = canvas.height - 30;

        // Ball velocity (speed and direction)
        var dx = 2;
        var dy = -2;

        // Ball size
        var ballRadius = 10;

        function draw() {
            // Clear the entire canvas
            ctx.clearRect(0, 0, canvas.width, canvas.height);

            // Draw the ball
            ctx.beginPath();
            ctx.arc(x, y, ballRadius, 0, Math.PI * 2);
            ctx.fillStyle = "#0095DD";
            ctx.fill();
            ctx.closePath();

            // Bounce off left and right walls
            if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) {
                dx = -dx;
            }

            // Bounce off top and bottom walls
            if (y + dy > canvas.height - ballRadius || y + dy < ballRadius) {
                dy = -dy;
            }

            // Move the ball
            x += dx;
            y += dy;
        }

        // Run draw() every 10 milliseconds
        setInterval(draw, 10);
    </script>
</body>
</html>

Code walkthrough / コードの解説

  • Lines 1-10: Standard HTML setup with a canvas element (480 pixels wide, 320 pixels tall). キャンバス要素を含む標準的なHTMLセットアップ。
  • Lines 11-12: Get references to the canvas and its 2D drawing context. キャンバスとその2D描画コンテキストへの参照を取得。
  • Lines 14-15: Set the ball's starting position (center-bottom of canvas). ボールの開始位置を設定（キャンバスの中央下部）。
  • Lines 17-18: Set velocity: dx=2 means 2 pixels right per frame, dy=-2 means 2 pixels up per frame. 速度を設定：dx=2はフレームごとに右に2ピクセル、dy=-2は上に2ピクセル。
  • Line 20: Ball radius of 10 pixels (ball is 20 pixels in diameter). ボールの半径10ピクセル（直径20ピクセル）。
  • Lines 22-24: Clear the canvas so old frames do not stack up (creates animation). キャンバスをクリアして古いフレームが重ならないようにする（アニメーションを作成）。
  • Lines 26-30: Draw the ball at its current (x, y) position. 現在の(x, y)位置にボールを描画。
  • Lines 32-34: Boundary detection for horizontal walls with radius correction. If hit, reverse dx. 半径補正付きの水平壁の境界検出。ぶつかったらdxを反転。
  • Lines 36-38: Boundary detection for vertical walls with radius correction. If hit, reverse dy. 半径補正付きの垂直壁の境界検出。ぶつかったらdyを反転。
  • Lines 40-41: Update the ball's position by adding velocity to current position. 速度を現在位置に足してボールの位置を更新。

Order matters / 順番が重要: Notice that we check boundaries BEFORE updating position (x += dx). This ensures the ball never actually moves past the wall. The check happens first, reversal happens if needed, then the position updates in the correct direction.

境界チェックは位置更新（x += dx）の前に行っていることに注目してください。これにより、ボールが実際に壁を超えることがありません。チェックが先に行われ、必要なら反転が起き、その後正しい方向に位置が更新されます。

14 Key Takeaways / 重要なポイント

1
The if statement lets code make decisions. It checks a condition and runs code only if that condition is true.
if文はコードに判断させます。条件をチェックし、その条件がtrueのときだけコードを実行します。
2
Comparison operators (<, >, <=, >=, ===, !==) compare two values and return true or false.
比較演算子は2つの値を比較し、trueまたはfalseを返します。
3
Boundary detection checks if the ball's next position will exceed the canvas boundaries.
境界検出は、ボールの次の位置がキャンバスの境界を超えるかどうかをチェックします。
4
Velocity reversal (dx = -dx) flips the direction of movement while keeping the speed the same.
速度の反転（dx = -dx）は、速さを同じに保ちながら動きの方向を反転します。
5
Always account for ball radius in boundary checks to prevent visual clipping through walls.
壁への視覚的な食い込みを防ぐため、境界チェックでは常にボールの半径を考慮しましょう。
6
Logical operators combine conditions: || (OR) means "at least one is true"; && (AND) means "both must be true."
論理演算子は条件を組み合わせます：||（OR）は「少なくとも1つがtrue」、&&（AND）は「両方がtrueでなければならない」。
7
Each axis is independent. Horizontal bouncing (dx) and vertical bouncing (dy) are handled separately.
各軸は独立しています。水平方向の跳ね返り（dx）と垂直方向の跳ね返り（dy）は別々に処理されます。
