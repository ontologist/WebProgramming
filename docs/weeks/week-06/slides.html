The Problem / 問題

The ball flies off the screen!

Right now, our ball moves in one direction forever. Once it reaches the edge of the canvas, it keeps going and disappears. We never see it again.

What we have: A ball that moves but escapes the canvas.

What we want: A ball that bounces off the walls -- like a real ball in a room!

To fix this, we need our code to make a decision: "Has the ball reached a wall? If yes, reverse its direction."

ボールが画面外に飛んでいく！

現在、ボールは一方向に永遠に動きます。キャンバスの端に到達すると、そのまま進んで消えてしまいます。

今あるもの： 動くがキャンバスから逃げるボール。

欲しいもの： 壁で跳ね返るボール -- 部屋の中の本物のボールのように！

これを修正するには、コードに判断させる必要があります：「ボールは壁に到達したか？はいなら、方向を反転する。」

Making Decisions in Code / コードで判断する

The if Statement

The if statement lets your code make decisions. It checks a condition and runs code only if that condition is true.

if (condition) { // this code runs ONLY when condition is true }

Real-life analogy:

  • "If it is raining, then bring an umbrella."
  • "If the ball hits a wall, then reverse direction."

if文（もし〜ならば）

if文はコードに判断させます。条件をチェックし、その条件が真（true）のときだけコードを実行します。

実生活のたとえ：

  • 「もし雨が降っているなら、傘を持っていく。」
  • 「もしボールが壁にぶつかったなら、方向を反転する。」

if Statement Syntax / if文の構文

if (condition) { // code to run when condition is true } // Example: var score = 10; if (score > 5) { console.log("High score!"); }

Parts of the if statement:

  • if -- the keyword that starts the decision
  • ( ) -- parentheses contain the condition to check
  • { } -- curly braces contain the code to run if true

Important: The condition inside ( ) must evaluate to true or false. This is called a boolean value.

重要： ( ) の中の条件は true か false に評価されなければなりません。これをブール値と呼びます。

if文の各部分：

  • if -- 判断を開始するキーワード
  • ( ) -- チェックする条件を含む括弧
  • { } -- 条件が真のとき実行するコードを含む波括弧

Comparison Operators / 比較演算子

Comparison operators let you compare two values. They always return true or false.

x > 10 // x is greater than 10 x < 10 // x is less than 10 x >= 10 // x is greater than or equal to 10 x <= 10 // x is less than or equal to 10 x === 10 // x is exactly equal to 10 x !== 10 // x is not equal to 10

Examples with real values:

var x = 7; x > 5 // true (7 is greater than 5) x < 3 // false (7 is not less than 3) x === 7 // true (7 equals 7) x !== 7 // false (7 does equal 7)

比較演算子

比較演算子は2つの値を比較します。常にtrueまたはfalseを返します。

  • > -- より大きい
  • < -- より小さい
  • >= -- 以上
  • <= -- 以下
  • === -- 等しい
  • !== -- 等しくない

Canvas Coordinates / キャンバスの座標系

Remember: the canvas coordinate system starts at the top-left corner (0, 0).

(0,0) -------- x increases -------> (canvas.width, 0) | | | | y increases | | | | | v v (0, canvas.height) -----------> (canvas.width, canvas.height)

Key boundaries:

  • Left wall: x = 0
  • Right wall: x = canvas.width
  • Top wall: y = 0
  • Bottom wall: y = canvas.height

キャンバスの座標系

覚えておこう：キャンバスの座標系は左上隅 (0, 0) から始まります。

  • 左壁： x = 0
  • 右壁： x = canvas.width
  • 上壁： y = 0
  • 下壁： y = canvas.height

Detecting the Right Wall / 右壁の検出

The ball hits the right wall when its x position exceeds the canvas width. We check where the ball will be after the next move:

// If the ball's next x position goes past the right wall... if (x + dx > canvas.width) { dx = -dx; // reverse horizontal direction! }

What happens step by step:

  1. x + dx calculates where the ball will be next frame
  2. If that position is greater than canvas.width, the ball has hit the right wall
  3. dx = -dx flips the horizontal velocity (e.g., 2 becomes -2)
  4. The ball now moves left instead of right!

右壁の検出

ボールのx位置がキャンバスの幅を超えると、右壁にぶつかります。次の移動後にボールがどこにいるかをチェックします。

  1. x + dx が次のフレームでのボールの位置を計算する
  2. その位置が canvas.width より大きければ、右壁に到達している
  3. dx = -dx で水平方向の速度を反転する（例：2が-2になる）
  4. ボールは右ではなく左に動くようになる！

Detecting the Left Wall / 左壁の検出

The ball hits the left wall when its x position goes below 0:

// If the ball's next x position goes past the left wall... if (x + dx < 0) { dx = -dx; // reverse horizontal direction! }

We can combine both checks:

if (x + dx > canvas.width || x + dx < 0) { dx = -dx; }

The || means "OR" -- if either condition is true, reverse direction.

左壁の検出

ボールのx位置が0を下回ると、左壁にぶつかります。

両方のチェックを組み合わせることができます。|| は「または」を意味します -- どちらかの条件が真なら、方向を反転します。

Top & Bottom Walls / 上壁と下壁

The same logic applies to vertical movement (y-axis):

// Top and bottom wall detection if (y + dy > canvas.height || y + dy < 0) { dy = -dy; // reverse vertical direction! }

Complete wall detection (4 walls):

// Horizontal walls (left and right) if (x + dx > canvas.width || x + dx < 0) { dx = -dx; } // Vertical walls (top and bottom) if (y + dy > canvas.height || y + dy < 0) { dy = -dy; }

Key insight: dx = -dx reverses horizontal movement. dy = -dy reverses vertical movement. Each axis is independent!

重要なポイント： dx = -dx は水平方向の動きを反転。dy = -dy は垂直方向の動きを反転。各軸は独立しています！

上壁と下壁の検出

同じロジックが垂直方向の動き（y軸）にも適用されます。

dx = -dx は水平の反転、dy = -dy は垂直の反転です。

Velocity Reversal: dx = -dx / 速度の反転

Why does dx = -dx work?

// If dx is 2 (moving right): dx = -dx; // dx becomes -2 (now moving left!) // If dx is -2 (moving left): dx = -dx; // dx becomes 2 (now moving right!) // The negative sign "flips" the direction.

Think of a ball bouncing off a wall in real life. When it hits, its speed stays the same, but the direction reverses. That is exactly what negating the velocity does in code.

Before bounce: dx = 2 (moving right at speed 2) After bounce: dx = -2 (moving left at speed 2) Before bounce: dy = -3 (moving up at speed 3) After bounce: dy = 3 (moving down at speed 3)

なぜ dx = -dx が機能するのか？

実生活でボールが壁に跳ね返るのを考えてください。ぶつかると、速さは同じままですが、方向が反転します。コードで速度を否定する（マイナスにする）のは、まさにそれと同じことです。

マイナス記号が方向を「反転」させます。

Accounting for Ball Radius / ボールの半径を考慮する

There is a problem: the ball clips into the wall before bouncing! This is because we check the center of the ball, not its edge.

// WRONG: checks center point only if (x + dx > canvas.width || x + dx < 0) { ... } // CORRECT: accounts for ball radius if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) { dx = -dx; } if (y + dy > canvas.height - ballRadius || y + dy < ballRadius) { dy = -dy; }

Why?

  • Right wall: ball edge = x + ballRadius, so bounce when x + dx > canvas.width - ballRadius
  • Left wall: ball edge = x - ballRadius, so bounce when x + dx < ballRadius
  • Same logic for top (ballRadius) and bottom (canvas.height - ballRadius)

ボールの半径を考慮する

問題があります：ボールが跳ね返る前に壁に食い込んでしまいます！これはボールの端ではなく中心をチェックしているためです。

ballRadiusを引いたり足したりして、ボールの端が壁に触れたときに跳ね返るようにします。

Logical Operators / 論理演算子

&& (AND) and || (OR)

// || means OR - true if EITHER condition is true if (x > 100 || y > 100) { // runs if x > 100 OR y > 100 (or both) } // && means AND - true only if BOTH conditions are true if (x > 0 && x < 480) { // runs only if x is between 0 and 480 }

How to remember:

  • || (OR) = "at least one must be true" -- like a door with two keys, either one opens it
  • && (AND) = "both must be true" -- like a safe that needs two keys at the same time

論理演算子

  • || (OR / または) = 「少なくとも1つが真であればよい」
  • && (AND / かつ) = 「両方とも真でなければならない」

覚え方：|| は「どちらかの鍵で開くドア」、&& は「2つの鍵が同時に必要な金庫」

The Complete Bouncing Ball / 完全な跳ね返るボール

var canvas = document.getElementById("myCanvas"); var ctx = canvas.getContext("2d"); var x = canvas.width / 2; var y = canvas.height - 30; var dx = 2; var dy = -2; var ballRadius = 10; function draw() { ctx.clearRect(0, 0, canvas.width, canvas.height); ctx.beginPath(); ctx.arc(x, y, ballRadius, 0, Math.PI * 2); ctx.fillStyle = "#0095DD"; ctx.fill(); ctx.closePath(); // Bounce off left and right walls if (x + dx > canvas.width - ballRadius || x + dx < ballRadius) { dx = -dx; } // Bounce off top and bottom walls if (y + dy > canvas.height - ballRadius || y + dy < ballRadius) { dy = -dy; } x += dx; y += dy; } setInterval(draw, 10);

完全な跳ね返るボールのコード

これがセッション6の最終成果物です。ボールが4つの壁すべてで跳ね返ります！
