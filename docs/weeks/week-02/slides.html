What is Semantic HTML? / セマンティックHTMLとは？

Semantic = Meaningful / セマンティック = 意味のある

Semantic HTML uses tags that describe the meaning of the content, not just how it looks.

セマンティックHTMLは、見た目だけでなくコンテンツの意味を説明するタグを使います。

Before HTML5 / HTML5以前:

<div id="header">...</div> <div id="navigation">...</div> <div id="main-content">...</div> <div id="footer">...</div>

With HTML5 Semantic Tags / HTML5セマンティックタグ:

<header>...</header> <nav>...</nav> <main>...</main> <footer>...</footer>

Why does this matter? Accessibility (screen readers), SEO (search engines), and readability (easier for developers to understand).

なぜ重要？ アクセシビリティ（スクリーンリーダー）、SEO（検索エンジン）、可読性（開発者が理解しやすい）。

The <header> Element / <header>要素

The <header> element represents the introductory content at the top of your page - typically a logo, site title, or banner.

<header> <h1>My Personal Website</h1> <p>Welcome to my page!</p> </header>

Important! / 重要！

<header> is NOT the same as <head>!

  • <head> = metadata (title, charset, CSS links) - invisible to users
  • <header> = visible page header (logo, site title) - users see this!

<header>要素について：

ページ上部の導入コンテンツ（ロゴ、サイトタイトル、バナーなど）を表します。

<head>（メタデータ、非表示）と<header>（ページヘッダー、表示される）は別物です！

The <nav> Element / <nav>要素

The <nav> element contains navigation links - the menu that lets users move between pages.

<nav> <ul> <li><a href="aboutme.htm">About Me</a></li> <li><a href="hobbies.htm">Hobbies</a></li> <li><a href="job.htm">Future Job</a></li> </ul> </nav>

<nav>要素について：

ナビゲーションリンクを含みます。ユーザーがページ間を移動するためのメニューです。

  • <ul>と<li>を使ってリンクをリストにまとめます
  • <a href="...">でリンク先のファイル名を指定します
  • 相対パス（同じフォルダ内のファイル名）を使います

The <main>, <section>, <article> Elements

<main> - Primary Content / メインコンテンツ

The main content of the page. Only one per page!

<section> - Thematic Grouping / テーマ別グループ

Groups related content together by theme or topic.

<article> - Self-contained Content / 独立コンテンツ

Content that makes sense on its own (blog post, news article).

<main> <section> <h2>About Me</h2> <article> <p>Hello! I am a student...</p> </article> </section> <section> <h2>My Skills</h2> <p>HTML, CSS, and more!</p> </section> </main>

3つの要素の違い：

  • <main> = ページの主要コンテンツ（1ページに1つだけ）
  • <section> = テーマ別にコンテンツをグループ化
  • <article> = それだけで意味が通じる独立したコンテンツ

The <footer> Element / <footer>要素

The <footer> element contains information at the bottom of the page - copyright, contact info, or additional links.

<footer> <p>&copy; 2026 Your Name. All rights reserved.</p> <p>Contact: yourname@example.com</p> </footer>

Tip / ヒント:

&copy; creates the copyright symbol: ©

&copy; は著作権記号 © を表示します。

<footer>要素について：

ページの下部に表示される情報（著作権、連絡先、追加リンクなど）を含みます。

すべてのページに一貫したフッターを入れましょう。

Putting It All Together / すべてをまとめる

Here is a complete semantic HTML5 page structure:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My Website</title> </head> <body> <header> <h1>My Website</h1> </header> <nav> <ul> <li><a href="aboutme.htm">About</a></li> <li><a href="hobbies.htm">Hobbies</a></li> <li><a href="job.htm">Job</a></li> </ul> </nav> <main> <section> <h2>Welcome!</h2> <p>This is my personal site.</p> </section> </main> <footer> <p>&copy; 2026 My Name</p> </footer> </body> </html>

セマンティックHTML5の完全なページ構造です。header、nav、main、footerが順番に並んでいます。

Multi-page Websites / 複数ページのウェブサイト

How It Works:

A multi-page website is simply multiple .html files in one folder, linked together with <a> tags.

Your Project Folder / プロジェクトフォルダ:

mysite/ aboutme.htm hobbies.htm job.htm photo.jpg

Linking Between Pages / ページ間のリンク:

<!-- On aboutme.htm --> <a href="hobbies.htm">My Hobbies</a> <a href="job.htm">My Future Job</a> <!-- On hobbies.htm --> <a href="aboutme.htm">About Me</a> <a href="job.htm">My Future Job</a>

相対パスでリンク：

同じフォルダ内のファイルは、ファイル名だけでリンクできます（例：href="hobbies.htm"）。

すべてのページに同じナビゲーションを入れて、一貫性を保ちましょう。

Assignment 1: Create Your Personal Website!

課題1：あなたの個人ウェブサイトを作ろう！

Requirements / 要件:

  • 3 pages / 3ページ:
    • aboutme.htm - Self-introduction / 自己紹介
    • hobbies.htm - Your hobbies / あなたの趣味
    • job.htm - Your dream job / あなたの将来の仕事
  • Use semantic HTML5 tags on every page
  • Navigation linking all 3 pages on every page
  • At least one image per page

すべてのページでセマンティックHTML5タグを使い、全ページにナビゲーションを配置し、各ページに最低1つの画像を入れてください。

MySite: aboutme.htm / 自己紹介ページ

Example structure for your About Me page:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>About Me</title> </head> <body> <header> <h1>Taro Yamada</h1> </header> <nav> <ul> <li><a href="aboutme.htm">About Me</a></li> <li><a href="hobbies.htm">Hobbies</a></li> <li><a href="job.htm">Future Job</a></li> </ul> </nav> <main> <section> <h2>About Me</h2> <img src="myphoto.jpg" alt="My photo"> <p>Hello! I am a university student...</p> </section> </main> <footer> <p>&copy; 2026 Taro Yamada</p> </footer> </body> </html>

header（名前）、nav（3ページへのリンク）、main（自己紹介と写真）、footer（著作権）の構造を使いましょう。

MySite: hobbies.htm & job.htm

Same Structure, Different Content! / 同じ構造、違うコンテンツ！

Every page uses the same semantic structure and the same navigation menu.

hobbies.htm - Key Differences:

<main> <section> <h2>My Hobbies</h2> <article> <h3>Soccer</h3> <img src="soccer.jpg" alt="Soccer"> <p>I have played soccer since...</p> </article> </section> </main>

job.htm - Key Differences:

<main> <section> <h2>My Future Job</h2> <img src="engineer.jpg" alt="Engineer"> <p>I want to become a software engineer because...</p> </section> </main>

ナビゲーションは全ページ共通にしましょう。ユーザーがどのページからでも他のページに移動できるようにします。

Key Takeaways / 重要なポイント

  • Semantic HTML = meaningful structure - Tags describe what the content IS, not how it looks
  • 6 key semantic elements: <header>, <nav>, <main>, <section>, <article>, <footer>
  • Linking pages with <a href="filename.htm"> using relative paths
  • Consistent navigation on every page for good user experience

まとめ：

  • セマンティックHTML = 意味のある構造 - タグがコンテンツの意味を表す
  • 6つの主要セマンティック要素： header, nav, main, section, article, footer
  • ページ間リンク： 相対パスで <a href="ファイル名.htm"> を使う
  • 一貫したナビゲーション： すべてのページに同じメニューを配置
