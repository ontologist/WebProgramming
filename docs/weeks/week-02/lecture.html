3. Why Semantic HTML Matters / セマンティックHTMLが重要な理由

The problem with <div> everywhere / <div>だらけの問題

Before HTML5, web developers used <div> tags for everything. A typical old webpage looked like this:
HTML5以前、ウェブ開発者はすべてに<div>タグを使用していました。典型的な古いウェブページは次のようでした：

<div id="header">
  <div id="nav">...</div>
</div>
<div id="main">
  <div id="article">...</div>
  <div id="sidebar">...</div>
</div>
<div id="footer">...</div>

The problem: The <div> tag has no meaning. It is just a generic container. The browser, search engines, and screen readers cannot tell what the content actually is.
問題点：<div>タグには意味がありません。単なる汎用的なコンテナです。ブラウザ、検索エンジン、スクリーンリーダーは、コンテンツが実際に何であるかを判断できません。

What does "semantic" mean? / 「セマンティック」とは？

Semantic means "relating to meaning." A semantic HTML element clearly describes its meaning to both the browser and the developer.
セマンティックとは「意味に関する」という意味です。セマンティックHTML要素は、ブラウザと開発者の両方にその意味を明確に伝えます。

  • <div> = "I am a box" (no meaning)
    「私はボックスです」（意味なし）
  • <nav> = "I contain navigation links" (clear meaning!)
    「私はナビゲーションリンクを含んでいます」（明確な意味！）
  • <article> = "I am a self-contained piece of content" (clear meaning!)
    「私は独立したコンテンツです」（明確な意味！）

Benefits of Semantic HTML / セマンティックHTMLの利点

  1. Accessibility (アクセシビリティ) — Screen readers can navigate your page more effectively. Visually impaired users depend on semantic elements to understand page structure.
    スクリーンリーダーがページをより効果的にナビゲートできます。視覚障がいのあるユーザーはページ構造を理解するためにセマンティック要素に依存しています。
  2. SEO (検索エンジン最適化) — Search engines like Google understand your content better, which helps your site rank higher in search results.
    Googleなどの検索エンジンがコンテンツをよりよく理解し、検索結果での上位表示に役立ちます。
  3. Readability (可読性) — Other developers (and future you!) can read and understand your code more easily.
    他の開発者（そして将来のあなた自身！）がコードをより簡単に読み理解できます。
  4. Maintainability (保守性) — Semantic code is easier to update and fix because each section has a clear purpose.
    各セクションに明確な目的があるため、セマンティックなコードは更新や修正が容易です。

4. Semantic HTML5 Elements — Detailed Reference / セマンティックHTML5要素 — 詳細リファレンス

Let's look at each semantic element in detail. For each one, you will see what it means, when to use it, and a code example.
各セマンティック要素を詳しく見ていきましょう。それぞれの意味、使用するタイミング、コード例を紹介します。

<header> — Introductory Content / ヘッダー — 導入コンテンツ

The <header> element represents introductory content at the top of a page or section. It typically contains the site name, logo, or main heading.
<header>要素は、ページまたはセクションの上部にある導入コンテンツを表します。通常、サイト名、ロゴ、またはメインの見出しを含みます。

Important: <header> is NOT the same as <head>!

  • <head> = invisible metadata (title, CSS links, etc.) — goes before <body>
  • <header> = visible content at the top of your page — goes inside <body>

<header>と<head>は異なります！<head>は非表示のメタデータで、<header>はページ上部の表示コンテンツです。

<header>
  <h1>My Personal Website</h1>
  <p>Welcome to my page!</p>
</header>

<nav> — Navigation Links / ナビゲーションリンク

The <nav> element contains your main navigation links. It usually holds a list of links (<ul> + <li> + <a>) that let visitors move between pages.
<nav>要素には、メインのナビゲーションリンクが含まれます。通常、訪問者がページ間を移動するためのリンクのリスト（ul + li + a）を保持します。

<nav>
  <ul>
    <li><a href="aboutme.htm">About Me</a></li>
    <li><a href="hobbies.htm">Hobbies</a></li>
    <li><a href="job.htm">Future Job</a></li>
  </ul>
</nav>

Tip: Screen readers look for <nav> to help users jump directly to navigation. Using <div> instead would make this impossible.

ヒント：スクリーンリーダーは<nav>を探して、ユーザーがナビゲーションに直接ジャンプできるようにします。

<main> — Dominant Content / メインコンテンツ

The <main> element wraps the primary content of your page — the content that is unique to this specific page. There should be only ONE <main> element per page.
<main>要素は、ページの主要コンテンツ（そのページに固有のコンテンツ）を囲みます。1ページにつき<main>要素は1つだけにしてください。

<main>
  <h2>About Me</h2>
  <p>Hello! My name is Tanaka Yuki. I am a university student
     studying policy studies.</p>
  <img src="myphoto.jpg" alt="Photo of Tanaka Yuki">
</main>

Rule: Only ONE <main> per page! Do not put your header, nav, or footer inside <main>.

ルール：<main>は1ページに1つだけ！ヘッダー、ナビ、フッターを<main>の中に入れないでください。

<section> — Thematic Grouping / セクション — テーマ別グループ

The <section> element groups related content together under a common theme. Each section typically has its own heading.
<section>要素は、共通のテーマの下に関連するコンテンツをグループ化します。各セクションには通常、独自の見出しがあります。

<main>
  <section>
    <h2>My Education</h2>
    <p>I graduated from Tokyo High School in 2024.</p>
  </section>

  <section>
    <h2>My Skills</h2>
    <ul>
      <li>HTML</li>
      <li>Microsoft Office</li>
    </ul>
  </section>
</main>

<article> — Self-contained Content / 記事 — 独立したコンテンツ

The <article> element represents content that is self-contained and could be independently distributed or reused — like a blog post, a news article, or a product card.
<article>要素は、独立して配布または再利用できる自己完結型のコンテンツを表します。ブログ記事、ニュース記事、製品カードなどです。

<article>
  <h2>My Favorite Book</h2>
  <p>I love reading "Norwegian Wood" by Haruki Murakami.
     It is a beautiful story about growing up.</p>
  <p>Published: April 2026</p>
</article>

How to decide: Ask yourself: "Does this content make sense on its own, outside of this page?" If yes, use <article>. If it is just a group of related content within the page, use <section>.

判断方法：「このコンテンツは、このページの外で単独で意味がありますか？」と自問してください。はいなら<article>、ページ内の関連コンテンツのグループなら<section>を使います。

<footer> — Footer Information / フッター情報

The <footer> element contains footer information such as copyright notices, contact details, or links to related pages. It appears at the bottom of your page.
<footer>要素には、著作権表示、連絡先、関連ページへのリンクなどのフッター情報が含まれます。ページの下部に表示されます。

<footer>
  <p>&copy; 2026 Tanaka Yuki. All rights reserved.</p>
  <p>Contact: yuki@example.com</p>
</footer>

Note: The &copy; code creates the © symbol. This is called an HTML entity.

注意：&copy;コードは©記号を作成します。これはHTMLエンティティと呼ばれます。

<aside> — Sidebar / Tangential Content / サイドバー / 補足コンテンツ

The <aside> element contains content that is tangentially related to the main content — like a sidebar, pull-quote, or additional information.
<aside>要素には、メインコンテンツに間接的に関連するコンテンツが含まれます。サイドバー、引用、追加情報などです。

<aside>
  <h3>Fun Fact</h3>
  <p>The first website was created by Tim Berners-Lee
     in 1991 at CERN, Switzerland.</p>
</aside>

5. Page Layout with Semantic Elements / セマンティック要素によるページレイアウト

Now let's see how all these semantic elements work together in a complete page. Study this annotated example carefully:
次に、これらすべてのセマンティック要素が完全なページでどのように連携するかを見てみましょう。この注釈付きの例を注意深く学習してください：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My Website</title>
</head>
<body>

  <!-- HEADER: Site branding and title -->
  <header>
    <h1>Tanaka Yuki's Website</h1>
    <p>Welcome to my personal site!</p>
  </header>

  <!-- NAV: Navigation links to other pages -->
  <nav>
    <ul>
      <li><a href="aboutme.htm">About Me</a></li>
      <li><a href="hobbies.htm">Hobbies</a></li>
      <li><a href="job.htm">Future Job</a></li>
    </ul>
  </nav>

  <!-- MAIN: The unique content of THIS page (only ONE per page!) -->
  <main>

    <!-- SECTION: A thematic group within the main content -->
    <section>
      <h2>About Me</h2>
      <p>I am a first-year student studying policy studies.</p>
      <img src="myphoto.jpg" alt="Photo of me">
    </section>

    <section>
      <h2>My Goals</h2>
      <p>I want to become a web developer.</p>
    </section>

    <!-- ASIDE: Extra info, not the main focus -->
    <aside>
      <h3>Did you know?</h3>
      <p>Over 1.9 billion websites exist on the internet!</p>
    </aside>

  </main>

  <!-- FOOTER: Copyright and contact info -->
  <footer>
    <p>&copy; 2026 Tanaka Yuki</p>
  </footer>

</body>
</html>

Visual structure / 視覚的な構造:

Think of the page like a newspaper:

  • <header> = the newspaper's masthead (name at the top)
  • <nav> = the table of contents
  • <main> = the main stories
  • <section> = different sections (sports, weather, etc.)
  • <article> = an individual news article
  • <aside> = the sidebar with ads or fun facts
  • <footer> = the small print at the bottom

ページを新聞のように考えてください：headerは新聞の題字、navは目次、mainはメイン記事、sectionは各セクション、articleは個々の記事、asideはサイドバー、footerは下部の小さな文字です。

6. Creating Multi-page Websites / 複数ページウェブサイトの作成

A real website usually has multiple pages. Let's learn how to create and link them together.
実際のウェブサイトには通常、複数のページがあります。それらを作成してリンクする方法を学びましょう。

File Organization / ファイルの整理

Keep all your .htm files in one folder. This makes linking between pages easy. Here is an example folder structure:
すべての.htmファイルを1つのフォルダに保管してください。これにより、ページ間のリンクが簡単になります。フォルダ構造の例：

MySite/
    aboutme.htm
    hobbies.htm
    job.htm
    myphoto.jpg
    hobby1.jpg
    hobby2.jpg

Relative Links / 相対リンク

When all your files are in the same folder, you link to them using just the filename:
すべてのファイルが同じフォルダにある場合、ファイル名だけでリンクできます：

<!-- Link to another page in the SAME folder -->
<a href="hobbies.htm">My Hobbies</a>

<!-- Link to an image in the SAME folder -->
<img src="myphoto.jpg" alt="My photo">

Important: Do not use full file paths like C:\Users\Yuki\Desktop\MySite\hobbies.htm. This will only work on your computer! Always use relative paths (just the filename).

重要：C:\Users\...のような完全なファイルパスを使わないでください。それはあなたのコンピューターでしか動作しません！常に相対パス（ファイル名のみ）を使用してください。

Why .htm vs .html? / .htmと.htmlの違い

Both .htm and .html file extensions work exactly the same way. Browsers treat them identically. In this course, we use .htm simply for brevity — it's slightly shorter to type. You can use either one, but be consistent: if you name a file hobbies.htm, make sure your links also say hobbies.htm, not hobbies.html.
.htmも.htmlも全く同じように動作します。ブラウザはどちらも同じに扱います。このコースでは、簡潔さのために.htmを使用します。どちらを使用しても構いませんが、一貫性を保ってください。

Navigation Consistency / ナビゲーションの一貫性

Every page in your site should have the same navigation menu. This lets visitors move between pages easily, no matter which page they are currently on.
サイトのすべてのページに同じナビゲーションメニューを配置してください。これにより、訪問者は現在どのページにいても簡単にページ間を移動できます。

<!-- Put this SAME nav on EVERY page -->
<nav>
  <ul>
    <li><a href="aboutme.htm">About Me</a></li>
    <li><a href="hobbies.htm">Hobbies</a></li>
    <li><a href="job.htm">Future Job</a></li>
  </ul>
</nav>

Tip: Copy and paste the <nav> section from your first page to every other page. This ensures all links are exactly the same.

ヒント：最初のページから<nav>セクションをコピーして他のすべてのページに貼り付けてください。これにより、すべてのリンクが完全に同じになります。

7. MySite Project — Step by Step Guide / MySiteプロジェクト — ステップバイステップガイド

Let's build your first multi-page website! Follow these steps carefully.
初めての複数ページウェブサイトを構築しましょう！以下のステップに注意深く従ってください。

Step 1: Create a folder called "MySite" / ステップ1：「MySite」フォルダを作成

On your Desktop (or Documents), create a new folder named MySite. All your project files will go inside this folder.
デスクトップ（またはドキュメント）に「MySite」という新しいフォルダを作成してください。すべてのプロジェクトファイルをこのフォルダに入れます。

Step 2: Create aboutme.htm / ステップ2：aboutme.htmを作成

Open VS Code, create a new file, and save it as aboutme.htm inside the MySite folder. Use the full semantic HTML5 structure:
VS Codeを開き、新しいファイルを作成して、MySiteフォルダ内にaboutme.htmとして保存します。完全なセマンティックHTML5構造を使用してください：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>About Me - MySite</title>
</head>
<body>

  <header>
    <h1>Tanaka Yuki's Website</h1>
  </header>

  <nav>
    <ul>
      <li><a href="aboutme.htm">About Me</a></li>
      <li><a href="hobbies.htm">Hobbies</a></li>
      <li><a href="job.htm">Future Job</a></li>
    </ul>
  </nav>

  <main>
    <section>
      <h2>About Me</h2>
      <img src="myphoto.jpg" alt="Photo of Tanaka Yuki">
      <p>Hello! My name is Tanaka Yuki.</p>
      <p>I am a first-year student at the School of Policy
         Studies. I am from Osaka, Japan.</p>
    </section>

    <section>
      <h2>My Interests</h2>
      <ul>
        <li>Web Development</li>
        <li>Photography</li>
        <li>Traveling</li>
      </ul>
    </section>
  </main>

  <footer>
    <p>&copy; 2026 Tanaka Yuki. All rights reserved.</p>
  </footer>

</body>
</html>

Step 3: Add your name, photo, and self-introduction / ステップ3：名前、写真、自己紹介を追加

Replace "Tanaka Yuki" with your own name. Add a photo of yourself (save the image in the MySite folder). Write a short self-introduction.
「Tanaka Yuki」を自分の名前に変更してください。自分の写真を追加し（MySiteフォルダに画像を保存）、短い自己紹介を書いてください。

Step 4: Create hobbies.htm / ステップ4：hobbies.htmを作成

Create a new file called hobbies.htm. Use the same layout structure (header, nav, main, footer). In the <main> section, write about your hobbies with images.
hobbies.htmという新しいファイルを作成してください。同じレイアウト構造（header、nav、main、footer）を使用します。mainセクションに趣味について画像付きで書いてください。

Step 5: Create job.htm / ステップ5：job.htmを作成

Create job.htm. Again, use the same structure. Write about your dream job or future career plans.
job.htmを作成してください。同じ構造を使用し、夢の仕事や将来のキャリアプランについて書いてください。

Step 6: Test all navigation links / ステップ6：すべてのナビゲーションリンクをテスト

Open aboutme.htm in your browser. Click every link in the navigation. Make sure you can move between all three pages without errors.
aboutme.htmをブラウザで開いてください。ナビゲーションのすべてのリンクをクリックし、エラーなく3つのページ間を移動できることを確認してください。

Common problem: If a link does not work, check the filename carefully. Hobbies.htm is not the same as hobbies.htm on some systems! Use lowercase filenames to be safe.

よくある問題：リンクが動作しない場合、ファイル名を注意深く確認してください。一部のシステムではHobbies.htmとhobbies.htmは異なります！安全のために小文字のファイル名を使用してください。

9. Common Mistakes to Avoid / よくある間違い

Here are the most common mistakes beginners make. Check your code for these issues!
初心者がよく犯す間違いです。コードにこれらの問題がないか確認してください！

Mistake 1: Forgetting to close tags / 間違い1：タグの閉じ忘れ

<!-- WRONG: missing closing tag -->
<p>Hello, my name is Yuki.
<p>I like programming.

<!-- CORRECT: every opening tag has a closing tag -->
<p>Hello, my name is Yuki.</p>
<p>I like programming.</p>

Mistake 2: Confusing <header> with <head> / 間違い2：<header>と<head>の混同

<!-- WRONG: putting visible content in <head> -->
<head>
  <h1>My Website</h1>  <!-- This will NOT display! -->
</head>

<!-- CORRECT: visible content goes in <header> inside <body> -->
<body>
  <header>
    <h1>My Website</h1>  <!-- This WILL display! -->
  </header>
</body>

Mistake 3: Broken links (wrong filename) / 間違い3：リンク切れ（ファイル名の間違い）

<!-- WRONG: filename doesn't match -->
<a href="Hobbies.html">Hobbies</a>
<!-- But your file is named "hobbies.htm" -->

<!-- CORRECT: exact match, including extension -->
<a href="hobbies.htm">Hobbies</a>

Remember: File names are case-sensitive on many systems. Hobbies.htm and hobbies.htm are two different files!

覚えておいてください：多くのシステムではファイル名は大文字と小文字が区別されます。

Mistake 4: Missing alt attributes on images / 間違い4：画像のalt属性の欠落

<!-- WRONG: no alt attribute -->
<img src="myphoto.jpg">

<!-- CORRECT: always include alt text -->
<img src="myphoto.jpg" alt="Photo of me at the park">

Why is alt important?

  • Screen readers read the alt text aloud to visually impaired users
  • If the image fails to load, the alt text is displayed instead
  • Search engines use alt text to understand your images

なぜaltが重要か：スクリーンリーダーがalt属性を読み上げます。画像が読み込めない場合にalt属性が表示されます。検索エンジンがalt属性を使って画像を理解します。

10. Key Takeaways / まとめ

  1. Semantic HTML gives meaning to your content. Use <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer> instead of <div> whenever possible.
    セマンティックHTMLはコンテンツに意味を与えます。可能な限り<div>の代わりにセマンティック要素を使用してください。
  2. Only ONE <main> per page. The header, nav, and footer go outside of main.
    1ページにつき<main>は1つだけ。header、nav、footerはmainの外に配置します。
  3. Keep navigation consistent. Every page should have the same menu so visitors can always find their way.
    ナビゲーションを一貫させてください。訪問者が常に移動できるように、すべてのページに同じメニューを配置します。
  4. Use relative links. When files are in the same folder, just use the filename: href="hobbies.htm".
    相対リンクを使用してください。ファイルが同じフォルダにある場合、ファイル名だけを使用します。
  5. Always include alt on images. This is essential for accessibility and good practice.
    画像には必ずalt属性を含めてください。アクセシビリティに不可欠で、良い習慣です。
  6. <header> is NOT <head>. Know the difference!
    <header>と<head>は異なります。違いを覚えてください！
