1. Assignment Overview / 課題概要

In this assignment, you will take the procedural ball code from your Breakout game and refactor it into a clean, reusable Ball class — without changing how the game looks or plays. This is your first real piece of object-oriented JavaScript.
この課題では、ブレイクアウトゲームの手続き型のボールコードを、きれいで再利用可能なBallクラスにリファクタリングします — ゲームの見た目や動作は変えずに。これがあなたの最初の本格的なオブジェクト指向JavaScriptです。

The game should look identical to the procedural version: the same yellow ball bouncing in the same canvas. But the code underneath will be much cleaner, and you will be ready to build more classes in Sessions 11–13.
ゲームは手続き型版と同じ見た目になります：同じ黄色いボールが同じキャンバスで跳ね返る。しかし、内部のコードはずっとクリーンになり、セッション11–13でさらに多くのクラスを作る準備が整います。

Goal: Demonstrate that you understand class syntax, the constructor(), the this keyword, methods, and creating objects with new.

目標：class構文、constructor()、thisキーワード、メソッド、newでのオブジェクト作成を理解していることを示す。

2. What You Need To Do / やるべきこと

Open your existing game.htm from Session 6 (or use the procedural version in Section 5 below as your starting point) and follow these steps:
セッション6の既存のgame.htmを開く（または、下のセクション5にある手続き型版をスタート地点として使う）か、以下のステップに従ってください：

Step 1: Define the Ball class / Ballクラスを定義する

Above your game code, write class Ball { ... }. By convention, the class name begins with a capital letter.
ゲームコードの上に、class Ball { ... }と書きます。慣習として、クラス名は大文字で始めます。

Step 2: Write the constructor() / constructor()を書く

Inside the class, write a constructor(startX, startY, radius). Assign every piece of ball state onto this: this.x, this.y, this.startX, this.startY (so you can reset later), this.radius, this.dx, this.dy, and this.color.
クラス内にconstructor(startX, startY, radius)を書きます。すべてのボール状態をthisに代入してください：this.x、this.y、this.startX、this.startY（後でリセットするため）、this.radius、this.dx、this.dy、this.color。

Step 3: Write the draw(ctx) method / draw(ctx)メソッドを書く

Copy the body of your old drawBall() function inside the class, but read every value from this (e.g., this.x instead of x). The only parameter should be ctx, the canvas drawing context.
古いdrawBall()関数の中身をクラス内にコピーし、すべての値をthisから読み取ります（例：xではなくthis.x）。唯一のパラメーターはctx、つまりキャンバス描画コンテキストにします。

Step 4: Write the move(canvasWidth, canvasHeight) method / move(canvasWidth, canvasHeight)メソッドを書く

Move the ball and bounce it off all four walls inside this method. Use this.x, this.y, this.dx, this.dy, and this.radius. The two parameters tell the method how big the canvas is.
このメソッド内でボールを動かし、4つの壁で跳ね返らせます。this.x、this.y、this.dx、this.dy、this.radiusを使います。2つのパラメーターはキャンバスのサイズをメソッドに伝えます。

Step 5: Write the reset() method / reset()メソッドを書く

Send this.x and this.y back to this.startX and this.startY, and restore this.dx and this.dy to their original values.
this.xとthis.yをthis.startXとthis.startYに戻し、this.dxとthis.dyを元の値に戻します。

Step 6: Create the ball with new / newでボールを作成する

Replace your old global variables (var x, var y, var dx, ...) with a single line: let ball = new Ball(240, 160, 10);.
古いグローバル変数（var x、var y、var dx、...）を1行に置き換えます：let ball = new Ball(240, 160, 10);。

Step 7: Use the methods in the game loop / ゲームループでメソッドを使う

Inside draw(), replace the old drawing and movement calls with ball.draw(ctx); and ball.move(canvas.width, canvas.height);. Delete the old drawBall() function and the loose x += dx; y += dy; lines.
draw()内で、古い描画と移動の呼び出しをball.draw(ctx);とball.move(canvas.width, canvas.height);に置き換えます。古いdrawBall()関数とバラバラのx += dx; y += dy;行を削除します。

Challenge (optional) / チャレンジ（任意）

From the same class, create two or three balls, each with a different starting position, size, and color (e.g., let ball2 = new Ball(100, 50, 15);). Call draw() and move() on each one in the loop. Notice you did not rewrite any ball logic!
同じクラスから、それぞれ異なる開始位置・サイズ・色の2〜3個のボールを作成します（例：let ball2 = new Ball(100, 50, 15);）。ループでそれぞれにdraw()とmove()を呼び出します。ボールのロジックを一切書き直していないことに気づくはずです！

5. Starting Point (Procedural Code) / スタート地点（手続き型コード）

</style>
</head>

<canvas id="myCanvas" width="480" height="320"></canvas>

<script>
const canvas = document.getElementById("myCanvas");
const ctx = canvas.getContext("2d");

// ===== Procedural ball state (these will all go away!) =====
var x = 240;
var y = 160;
var dx = 2;
var dy = -2;
var radius = 10;

function drawBall(x, y, radius) {
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, Math.PI * 2);
    ctx.fillStyle = "yellow";
    ctx.strokeStyle = "blue";
    ctx.fill();
    ctx.stroke();
    ctx.closePath();
}

function draw() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    drawBall(x, y, radius);
    // bounce off walls
    if (x + dx + radius > canvas.width || x + dx - radius < 0) {
        dx = -dx;
    }
    if (y + dy - radius < 0 || y + dy + radius > canvas.height) {
        dy = -dy;
    }
    x += dx;
    y += dy;
}

setInterval(draw, 10);
</script>

</body>
</html>

7. Tips for Success / 成功のためのヒント

  • Refactor in small steps. After each step (constructor, then draw, then move, ...), reload the browser and confirm nothing broke before moving on.
    小さなステップでリファクタリングしましょう。各ステップ（コンストラクター、次にdraw、次にmove...）の後、ブラウザを再読み込みして何も壊れていないことを確認してから次に進みます。
  • Inside any method, if you write x or dx by itself, that is almost always wrong — it should be this.x or this.dx.
    メソッド内でxやdxを単独で書いたら、ほぼ間違いなく誤りです — this.xやthis.dxであるべきです。
  • Use console.log(ball) in the browser console (F12) to inspect your ball object and see all of its current properties at once.
    ブラウザコンソール（F12）でconsole.log(ball)を使ってボールオブジェクトを調べ、現在のすべてのプロパティを一度に確認しましょう。
  • If the ball stops moving, you probably forgot the this.x += this.dx; lines at the bottom of move().
    ボールが動かなくなったら、おそらくmove()の最後のthis.x += this.dx;行を忘れています。
  • If the ball escapes the canvas, double-check that the wall-bounce conditions still use this.radius on both sides (top/bottom and left/right).
    ボールがキャンバスから出てしまったら、壁の跳ね返り条件でthis.radiusを両方の側（上/下と左/右）で使っているか再確認しましょう。
  • Once the single ball works, try the Challenge: let ball2 = new Ball(100, 50, 15); and call ball2.draw(ctx); ball2.move(...) in the loop. Watch the payoff of OOP appear with zero extra logic.
    1つのボールが動いたらチャレンジを試してみましょう：let ball2 = new Ball(100, 50, 15);とループ内でball2.draw(ctx); ball2.move(...)を呼びます。追加ロジックゼロでOOPの見返りが現れます。

8. Common Mistakes to Avoid / よくある間違い

  • Forgetting this. inside methods — x = x + dx does NOT update the object's property.
    メソッド内でthis.を忘れる — x = x + dxではオブジェクトのプロパティは更新されません。
  • Forgetting new — let ball = Ball(240, 160, 10); will throw an error or produce undefined.
    newを忘れる — let ball = Ball(240, 160, 10);はエラーになるかundefinedを返します。
  • Putting commas between methods in a class — class methods are separated by whitespace only, not commas.
    クラス内のメソッド間にカンマを置く — クラスメソッドは空白だけで区切ります、カンマではありません。
  • Naming the class with a lowercase letter (class ball) — works, but violates convention and will lose style points.
    クラスを小文字で命名する（class ball） — 動くが慣習に反し、スタイル点を失います。
  • Leaving the old var x = 240; var y = 160; globals in place. The game may still run, but the refactor is incomplete — the data must live on the ball object.
    古いvar x = 240; var y = 160;グローバルを残したままにする。ゲームは動くかもしれませんが、リファクタリングは未完了です — データはボールオブジェクトの上にある必要があります。
  • Calling ball.draw() without the ctx argument — the method needs the drawing context.
    ctx引数なしでball.draw()を呼び出す — メソッドは描画コンテキストを必要とします。
  • Calling ball.move() without the canvas dimensions — the ball cannot bounce if it does not know how big the canvas is.
    キャンバス寸法なしでball.move()を呼び出す — キャンバスのサイズがわからないとボールは跳ね返れません。
