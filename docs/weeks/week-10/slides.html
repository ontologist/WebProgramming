The Problem: Scattered Variables

問題：散らばった変数

Our Ball Today

var x = 240; var y = 160; var dx = 2; var dy = -2; var radius = 10; function drawBall(x, y, radius) { ... }

Why is this a problem?

  • These variables all belong to "the ball" — but nothing groups them
  • Want a 2nd ball? Copy x2, y2, dx2... — messy!
  • Data (top) is far from behavior (the draw loop)

なぜ問題なのか？

  • これらはすべて「ボール」に属するが、グループ化されていない
  • 2つ目のボールが欲しい？x2, y2, dx2...をコピー — 大混乱！
  • データ（上部）が振る舞い（描画ループ）から離れている

解決策：ボールのデータと振る舞いを1つにまとめる。

What is OOP?

OOPとは？

Object-Oriented Programming

Organize code around objects — units that bundle together:

  • Properties (data) — what the object knows (position, speed, size)
  • Methods (behavior) — what the object can do (draw, move, reset)

A real ball: knows its color, size, and speed (properties); can roll, bounce, stop (methods). We model the game ball the same way.

オブジェクト指向プログラミング

オブジェクトを中心にコードを整理します。オブジェクトは以下をまとめます：

  • プロパティ（データ） — オブジェクトが知っていること（位置、速度、サイズ）
  • メソッド（振る舞い） — オブジェクトができること（描画、移動、リセット）

本物のボールと同じようにゲームのボールをモデル化します。

Class vs Object

クラス対オブジェクト

Blueprint vs Building

Class = Blueprint

  • A plan on paper
  • A cookie cutter
  • Not a real ball
class Ball { ... }

Object = Instance

  • A real house
  • Each cookie stamped out
  • A real, usable ball
let ball = new Ball(...)

One blueprint → as many objects as you want!

設計図対建物

  • クラス = 設計図 — 紙の上の計画、クッキー型。本物のボールではない。
  • オブジェクト = インスタンス — 設計図から作られた本物のボール。

1つの設計図 → 好きなだけオブジェクトを作れる！

Class Syntax & the Constructor

クラス構文とコンストラクター

class Ball { constructor(x, y, radius) { this.x = x; this.y = y; this.radius = radius; this.dx = 2; this.dy = -2; this.color = "yellow"; } }
  • class keyword + Capital name
  • constructor() runs automatically when you create an object
  • It sets up the object's starting properties
  • classキーワード + 大文字の名前
  • constructor()はオブジェクト作成時に自動的に実行
  • オブジェクトの初期プロパティを設定する

The this Keyword

thisキーワード

"This specific object, right now"

this is how an object refers to its own properties.

Classroom analogy: when a student writes "my name", "my" means that student — not the whole class. this = this one object.

// ballA and ballB each have their OWN this.x ballA.move(); // changes ballA's this.x ballB.move(); // changes ballB's this.x

Watch out: x = x + dx does nothing — you must write this.x = this.x + this.dx.

「今、この特定のオブジェクト」

thisはオブジェクトが自分自身のプロパティを参照する方法です。

教室のたとえ：「私の名前」の「私の」はその生徒を指す。this = このオブジェクト1つ。

注意：クラス内では必ずthis.を付ける。

Methods vs Functions

メソッドと関数

A method is a function that belongs to an object

Function (before)

drawBall(x, y, radius);

You pass the data in.

Method (after)

ball.draw();

The ball already knows its own data via this.

Call a method with a dot: object.method()

メソッド = オブジェクトに属する関数

  • 関数（変更前）：drawBall(x, y, radius) — データを渡す
  • メソッド（変更後）：ball.draw() — ボールがthisで自分のデータを知っている

ドットで呼ぶ：object.method()

Creating Objects with new

newでオブジェクトを作成

Build a real object from the blueprint

// new runs the constructor let ball = new Ball(240, 160, 10); console.log(ball.x); // 240 console.log(ball.color); // "yellow" // Make MORE balls — same blueprint! let ball2 = new Ball(100, 50, 15); let ball3 = new Ball(400, 200, 8);

3 lines → 3 independent balls, each with its own position, speed, and size. No copy-pasting!

設計図から本物のオブジェクトを作る

newはコンストラクターを実行し、新しいオブジェクトを返します。

3行で3つの独立したボール — それぞれ独自の位置・速度・サイズ。コピペ不要！

The Ball Class: draw() & move()

Ballクラス：draw() と move()

Reads from this instead of parameters

draw(ctx) { ctx.beginPath(); ctx.arc(this.x, this.y, this.radius, 0, Math.PI*2); ctx.fillStyle = this.color; ctx.fill(); ctx.closePath(); } move(w, h) { if (this.x + this.dx + this.radius > w || this.x + this.dx - this.radius < 0) this.dx = -this.dx; if (this.y + this.dy - this.radius < 0 || this.y + this.dy + this.radius > h) this.dy = -this.dy; this.x += this.dx; this.y += this.dy; }

thisから読み取る

  • draw(ctx) — 自身のプロパティでボールを描画
  • move(w, h) — 4つの壁で跳ね返り、位置を更新（セッション6のロジック）

すべてボール自身の中で処理されます。

The Ball Class: reset()

Ballクラス：reset()

Return the ball to its start

In the constructor, we saved this.startX and this.startY. So resetting is easy:

constructor(startX, startY, radius) { this.x = startX; this.y = startY; this.startX = startX; // remember! this.startY = startY; ... } reset() { this.x = this.startX; this.y = this.startY; this.dx = 2; this.dy = -2; }

Call ball.reset() when the player loses a life.

ボールを開始位置に戻す

コンストラクターでthis.startXとthis.startYを保存したので、リセットは簡単です。

プレイヤーがライフを失ったときball.reset()を呼びます。

Before vs After

変更前と変更後

Before (procedural)

var x = 240, y = 160; var dx = 2, dy = -2; var radius = 10; function drawBall(...) {...} function draw() { ctx.clearRect(...); drawBall(x, y, radius); // 4 wall if-statements x += dx; y += dy; }

After (OOP)

let ball = new Ball(240, 160, 10); function draw() { ctx.clearRect(...); ball.draw(ctx); ball.move( canvas.width, canvas.height); }

Read it aloud: "ball draw, ball move." The code explains itself!

コードが文章のように読める

「ボール、描画。ボール、移動。」 — これが良いOOPの力です。手続き型の散らばったコードが、1つのきれいなオブジェクトにまとまりました。
