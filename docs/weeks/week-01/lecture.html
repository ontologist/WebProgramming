4 How the Web Works / Webの仕組み

Before we start writing code, let's understand what happens when you visit a website. It is simpler than you might think.

コードを書き始める前に、ウェブサイトにアクセスしたときに何が起こるかを理解しましょう。思っているより簡単です。

The Client-Server Model / クライアント・サーバーモデル

The Web works like a conversation between two computers:

  • Client (クライアント): Your web browser (Chrome, Firefox, Safari, Edge). It asks for web pages.
  • Server (サーバー): A computer somewhere on the internet that stores web pages and sends them when asked.
How a Web Page Loads / Webページの読み込みの仕組み
  Your Browser                          Web Server
  (Client)                              (Server)
  ┌──────────┐                       ┌──────────┐
  │          │  1. "Give me the      │          │
  │  Chrome  │ ──── page, please" ──>│  Server  │
  │  Firefox │     (HTTP Request)    │  stores  │
  │  Safari  │                       │  files   │
  │          │  2. "Here is the      │          │
  │          │ <── HTML file" ───────│          │
  │          │     (HTTP Response)   │          │
  └──────────┘                       └──────────┘
       │
       │ 3. Browser reads the HTML
       │    and displays the page
       ▼
  ┌──────────────┐
  │  You see the │
  │  web page!   │
  └──────────────┘

URLs: Addresses on the Web / URL：Webのアドレス

Every web page has an address called a URL (Uniform Resource Locator). Just like every house has a street address, every page has a URL.

すべてのWebページにはURL（Uniform Resource Locator）というアドレスがあります。すべての家に住所があるように、すべてのページにURLがあります。

Example: https://www.example.com/about.html

Part Example Meaning / 意味
Protocol https:// The language for communication (secure HTTP) / 通信のための言語（安全なHTTP）
Domain www.example.com The name of the server (like a street name) / サーバーの名前（通りの名前のようなもの）
Path /about.html Which specific page you want (like a house number) / 欲しい具体的なページ（番地のようなもの）

HTTP: The Language of the Web / HTTP：Webの言語

HTTP (HyperText Transfer Protocol) is the set of rules that browsers and servers use to talk to each other. Think of it as the language they both speak. When your browser wants a page, it sends an HTTP request. The server answers with an HTTP response containing the HTML file.

HTTP（HyperText Transfer Protocol）は、ブラウザとサーバーが通信するためのルールです。両者が話す言語だと考えてください。ブラウザがページを欲しいとき、HTTPのリクエストを送ります。サーバーはHTMLファイルを含むHTTPのレスポンスで答えます。

Simple Version / 簡単に言うと: You type a URL → your browser asks a server → the server sends back HTML → your browser shows the page. That's it!

URLを入力する → ブラウザがサーバーに問い合わせる → サーバーがHTMLを返す → ブラウザがページを表示する。それだけです！

5 What is HTML? / HTMLとは？

HTML stands for HyperText Markup Language.

HTMLはHyperText Markup Language（ハイパーテキスト・マークアップ・ランゲージ）の略です。

Let's break that name down:

  • HyperText — Text that can link to other text. When you click a link on a web page, that is hypertext in action. / 他のテキストにリンクできるテキスト
  • Markup — A way to "mark up" or annotate text to give it meaning. Like a teacher marking up a student's essay with red pen. / テキストに意味を与えるための注釈方法
  • Language — A set of rules and vocabulary (tags) that browsers understand. / ブラウザが理解するルールと語彙（タグ）のセット

A Brief History / 簡単な歴史

HTML was created by Tim Berners-Lee in 1991 at CERN (the European physics lab in Switzerland). He wanted scientists to share research documents with each other over the internet. His invention — the World Wide Web — changed the world forever.

HTMLは1991年にスイスのCERN（欧州物理学研究所）でティム・バーナーズ＝リーによって作られました。彼は科学者がインターネットを通じて研究文書を共有できるようにしたいと考えました。彼の発明 — ワールド・ワイド・ウェブ — は世界を永遠に変えました。

Today, HTML is the foundation of every single web page on the internet. Every website you have ever visited — Google, YouTube, Amazon, Twitter — is built with HTML.

HTML is NOT a Programming Language / HTMLはプログラミング言語ではない

Important distinction / 重要な区別: HTML is a markup language, not a programming language. It describes the structure and content of a page, but it cannot do calculations, make decisions, or respond to user actions. For that, we need JavaScript (which we will learn later in the course).

HTMLはマークアップ言語であり、プログラミング言語ではありません。ページの構造と内容を記述しますが、計算したり、判断したり、ユーザーの操作に反応したりすることはできません。それにはJavaScriptが必要です（コースの後半で学びます）。

Analogy / たとえ話: Think of HTML like the blueprint of a house. The blueprint tells you where the walls, doors, and windows go, but the blueprint itself is not the house. Similarly, HTML tells the browser where to put text, images, and links, but HTML itself is not a program.

HTMLは家の設計図のようなものです。設計図は壁、ドア、窓の位置を示しますが、設計図そのものは家ではありません。同様に、HTMLはブラウザにテキスト、画像、リンクの配置を指示しますが、HTML自体はプログラムではありません。

6 HTML Document Structure / HTML文書の構造

Every HTML page follows the same basic structure. Think of it as a template that every web page must use. Let's look at the simplest possible HTML document:

すべてのHTMLページは同じ基本構造に従います。すべてのWebページが使わなければならないテンプレートだと考えてください。最もシンプルなHTML文書を見てみましょう：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first web page.</p>
</body>
</html>

Now let's understand each part:

<!DOCTYPE html>

This line tells the browser: "This document is written in HTML5" (the latest version of HTML). It must always be the very first line of your file. It is not an HTML tag — it is an instruction to the browser.

この行はブラウザに「この文書はHTML5（HTMLの最新バージョン）で書かれています」と伝えます。常にファイルの最初の行でなければなりません。HTMLタグではなく、ブラウザへの指示です。

<html lang="en">

This is the root element — the container for everything on the page. All other HTML goes inside this tag. The lang="en" part tells the browser (and screen readers for visually impaired users) that the page is in English.

これはルート要素 — ページ上のすべてを入れるコンテナです。他のすべてのHTMLはこのタグの中に入ります。lang="en"は、ブラウザ（および視覚障害者用のスクリーンリーダー）に、ページが英語であることを伝えます。

<head>

The <head> section contains metadata — information about the page that is not displayed on screen. Think of it as the "backstage" area. It includes:

  • <meta charset="UTF-8"> — Tells the browser to use UTF-8 character encoding (which supports Japanese, emojis, and characters from every language). / UTF-8文字エンコーディングを使用（日本語、絵文字に対応）
  • <meta name="viewport"...> — Makes the page look correct on mobile phones. / スマートフォンでの正しい表示を確保
  • <title> — The text that appears in the browser tab. / ブラウザのタブに表示されるテキスト

<body>

The <body> section contains everything that users see on the page: text, images, links, buttons, videos, and more. This is the "stage" where your content performs.

<body>セクションには、ユーザーがページ上で見るすべてのものが含まれます：テキスト、画像、リンク、ボタン、動画など。これはコンテンツが表示される「ステージ」です。

Analogy / たとえ話: Think of an HTML document like a letter in an envelope:

  • <!DOCTYPE html> = the type of letter (official document) / 書類の種類
  • <html> = the envelope itself / 封筒そのもの
  • <head> = the information written on the outside of the envelope (return address, stamps) / 封筒の外側に書かれた情報
  • <body> = the letter inside the envelope (what you actually read) / 封筒の中の手紙（実際に読む内容）

Tags, Elements, and Nesting / タグ、要素、入れ子

HTML uses tags to mark up content. Most tags come in pairs:

<tagname>content goes here</tagname>
 ▲ opening tag               ▲ closing tag (has a /)

An element is the opening tag + content + closing tag together. Elements can be nested inside other elements, like boxes inside boxes:

<body>
    <h1>Welcome</h1>      <!-- h1 is inside body -->
    <p>Hello <strong>world</strong></p>  <!-- strong is inside p, which is inside body -->
</body>

Important / 重要: Always close your tags! If you open a <p>, you must close it with </p>. Forgetting to close tags is the most common beginner mistake.

タグは必ず閉じましょう！<p>を開いたら、</p>で閉じなければなりません。タグの閉じ忘れは初心者に最も多いミスです。

7 Essential HTML Tags / 基本的なHTMLタグ

Now let's learn the most important HTML tags. These are the building blocks you will use on almost every page.

では、最も重要なHTMLタグを学びましょう。これらはほぼすべてのページで使う基本要素です。

Headings: <h1> to <h6> / 見出し

Headings create titles and subtitles on your page. There are 6 levels, from <h1> (biggest/most important) to <h6> (smallest/least important).

見出しはページにタイトルとサブタイトルを作ります。<h1>（最大/最重要）から<h6>（最小/最少重要）まで6段階あります。

<h1>Main Title (biggest)</h1>
<h2>Section Title</h2>
<h3>Sub-section Title</h3>
<h4>Sub-sub-section Title</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Best Practice / ベストプラクティス: Use only one <h1> per page (the main title). Use <h2> for major sections, <h3> for subsections, and so on. Think of it like a book: one title, several chapter headings, many subheadings.

ページごとに<h1>は1つだけ使いましょう（メインタイトル）。主要セクションには<h2>、サブセクションには<h3>を使います。本のように考えましょう：タイトル1つ、章の見出しがいくつか、小見出しがたくさん。

Paragraphs: <p> / 段落

The <p> tag creates a paragraph of text. Browsers automatically add space above and below paragraphs.

<p>This is a paragraph. It can contain as much text as you want.
The browser will wrap the text automatically.</p>

<p>This is another paragraph. Notice the space between them.</p>

Note / 注意: Even if you press Enter many times in your HTML code, the browser will ignore extra whitespace. To start a new paragraph, you must use a new <p> tag.

HTMLコードで何度Enterを押しても、ブラウザは余分な空白を無視します。新しい段落を始めるには、新しい<p>タグを使う必要があります。

Emphasis: <strong> and <em> / 強調

<strong> makes text bold (strong importance). <em> makes text italic (emphasis).

<p>This word is <strong>very important</strong>.</p>
<p>This word has <em>emphasis</em>.</p>
<p>You can <strong><em>combine them</em></strong> too.</p>

Line Break: <br> and Horizontal Rule: <hr> / 改行と水平線

<br> forces a line break (like pressing Enter). <hr> creates a horizontal line to separate sections. Both are self-closing tags — they do not need a closing tag.

<p>Line one.<br>Line two (same paragraph, forced new line).</p>

<hr>

<p>This content is below the horizontal line.</p>

Lists: <ul>, <ol>, <li> / リスト

HTML gives you two types of lists:

  • <ul> — Unordered list (bullet points) / 順序なしリスト（箇条書き）
  • <ol> — Ordered list (numbered) / 順序ありリスト（番号付き）
  • <li> — List item (goes inside <ul> or <ol>) / リスト項目
<!-- Unordered list (bullets) -->
<ul>
    <li>Apples</li>
    <li>Bananas</li>
    <li>Cherries</li>
</ul>

<!-- Ordered list (numbers) -->
<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

What the unordered list looks like in the browser:

  • Apples
  • Bananas
  • Cherries

What the ordered list looks like in the browser:

  1. First step
  2. Second step
  3. Third step

Links: <a> / リンク

The <a> tag (short for "anchor") creates a clickable link to another page or website. The href attribute specifies the destination URL.

<a>タグ（「anchor」の略）は、別のページやウェブサイトへのクリック可能なリンクを作ります。href属性でリンク先のURLを指定します。

<!-- Link to an external website -->
<a href="https://www.google.com">Visit Google</a>

<!-- Link to another page on your site -->
<a href="about.html">About Me</a>

<!-- Link that opens in a new tab -->
<a href="https://www.google.com" target="_blank">Open Google in new tab</a>
Attribute Purpose / 目的 Example
href The URL or file path to link to / リンク先のURLまたはファイルパス href="about.html"
target Where to open the link / リンクを開く場所 target="_blank" (new tab)

Images: <img> / 画像

The <img> tag displays an image on the page. It is a self-closing tag (no closing tag needed). It requires two important attributes:

<img>タグはページに画像を表示します。自己閉鎖タグ（閉じタグ不要）です。2つの重要な属性が必要です：

<img src="photo.jpg" alt="A photo of Mount Fuji">
Attribute Purpose / 目的 Required?
src The file path or URL of the image / 画像のファイルパスまたはURL Yes
alt Alternative text shown if image fails to load, and read by screen readers for accessibility / 画像が読み込めない場合に表示されるテキスト、アクセシビリティのためにスクリーンリーダーが読み上げる Yes (always include!)

Always include alt text! / altテキストは必ず含めましょう！ It helps visually impaired users who use screen readers, and it is displayed if the image fails to load. It is also required by web accessibility standards.

8 Setting Up Your Development Environment / 開発環境のセットアップ

To write HTML, you need a text editor. We will use Visual Studio Code (VS Code), the most popular free code editor in the world.

HTMLを書くにはテキストエディタが必要です。世界で最も人気のある無料コードエディタ、Visual Studio Code（VS Code）を使います。

Step 1: Install VS Code / ステップ1：VS Codeをインストール

  • Go to https://code.visualstudio.com/
  • Click the download button for your operating system (Windows, Mac, or Linux)
  • Run the installer and follow the instructions
  • Open VS Code after installation is complete

Why VS Code?

  • Free — It costs nothing / 無料です
  • Powerful — Professional developers use it every day / プロの開発者が毎日使っています
  • Extensions — You can add extra features (we will install one next) / 拡張機能を追加できます
  • Syntax highlighting — It colors your code to make it easier to read / コードに色が付いて読みやすくなります
  • Auto-complete — It suggests code as you type / 入力中にコードを提案してくれます

Step 2: Install the Live Server Extension / ステップ2：Live Server拡張機能をインストール

Live Server is an extension that automatically refreshes your browser whenever you save your HTML file. This saves you from manually refreshing the page every time you make a change.

Live Serverは、HTMLファイルを保存するたびに自動的にブラウザを更新する拡張機能です。変更のたびに手動でページを更新する手間が省けます。

  • In VS Code, click the Extensions icon on the left sidebar (it looks like four squares)
  • In the search box, type Live Server
  • Find "Live Server" by Ritwick Dey and click Install
  • After installing, you will see a "Go Live" button at the bottom of VS Code

Step 3: Create Your First HTML File / ステップ3：最初のHTMLファイルを作成

  • Open VS Code
  • Click File → New File (or press Ctrl+N / Cmd+N on Mac)
  • Click File → Save As (or press Ctrl+S / Cmd+S on Mac)
  • Choose a location on your computer, name the file index.html, and click Save
  • Type the basic HTML template (from Section 6) and save the file

VS Code Shortcut / VS Codeのショートカット: In a new .html file, type ! (exclamation mark) and press Tab. VS Code will automatically generate the full HTML template for you!

新しい.htmlファイルで!（感嘆符）を入力してTabを押すと、VS Codeが自動的にHTMLテンプレートを生成してくれます！

Step 4: Open Your Page in a Browser / ステップ4：ブラウザでページを開く

There are two ways to see your page:

Method A: Double-click the file

  • Find your index.html file in your computer's file explorer
  • Double-click it — it will open in your default browser

Method B: Use Live Server (recommended)

  • With your index.html open in VS Code, click the "Go Live" button in the bottom-right corner
  • Your browser will open automatically with your page
  • Every time you save the file, the browser will refresh automatically

9 Browser Developer Tools / ブラウザ開発者ツール

Every modern browser has built-in Developer Tools (DevTools) that let you peek "behind the scenes" of any web page. These tools are essential for understanding and debugging your code.

すべてのモダンブラウザには、Webページの「舞台裏」を覗ける開発者ツール（DevTools）が組み込まれています。これらのツールはコードの理解とデバッグに不可欠です。

How to Open Developer Tools / 開発者ツールの開き方

Method / 方法 How / やり方
Keyboard shortcut Press F12 (works in all browsers)
Keyboard shortcut (Mac) Press Cmd + Option + I
Right-click menu Right-click anywhere on a page → select "Inspect" or "Inspect Element" / ページ上で右クリック → 「検証」を選択
Browser menu Menu → More Tools → Developer Tools

The Elements Tab / 要素タブ

The Elements tab shows you the HTML structure of the page as a tree. You can:

  • See all the HTML tags that make up the page / ページを構成するすべてのHTMLタグを見る
  • Click on any element to see it highlighted on the page / 要素をクリックするとページ上でハイライトされる
  • Edit HTML live — double-click on text in the Elements tab to change it (the changes are temporary and disappear when you refresh) / HTMLをライブ編集 — 要素タブのテキストをダブルクリックして変更（変更は一時的でリフレッシュすると消える）

Try this now! / 今すぐ試してみよう！ Open any website (like Google), press F12, and look at the Elements tab. You will see all the HTML that makes up that page. Try changing some text!

好きなウェブサイト（Googleなど）を開いてF12を押し、要素タブを見てみましょう。ページを構成するすべてのHTMLが表示されます。テキストを変更してみましょう！

The Console Tab / コンソールタブ

The Console tab is where you can type JavaScript commands and see messages from your code. We will use this extensively later in the course when we learn JavaScript. For now, just know it exists.

コンソールタブでは、JavaScriptコマンドを入力したり、コードからのメッセージを確認したりできます。JavaScriptを学ぶコース後半で頻繁に使います。今はその存在を知っておくだけで十分です。

Why DevTools Matter / DevToolsが重要な理由

  • Debugging: When something looks wrong, DevTools help you find the problem / 何かがおかしいとき、DevToolsが問題を見つける手助けをしてくれる
  • Learning: You can inspect any website to see how it was built / どんなウェブサイトでも検証して、どう作られたかを見ることができる
  • Experimenting: You can try changes without breaking your real code / 本物のコードを壊さずに変更を試すことができる

10 Practice Exercise / 練習問題

Let's put everything together! Follow these steps to create your first web page.

すべてをまとめましょう！以下の手順に従って、最初のWebページを作成してください。

Step-by-Step Instructions / ステップバイステップの手順

  • Create a folder on your Desktop (or any location you prefer) called MySite. デスクトップ（または好きな場所）にMySiteというフォルダを作成する
  • Open VS Code and go to File → Open Folder → select your MySite folder. VS Codeを開き、ファイル → フォルダを開く → MySiteフォルダを選択
  • Create a new file called index.html inside the MySite folder. MySiteフォルダ内にindex.htmlという新しいファイルを作成
  • Type the following code into your index.html file (type it yourself — do not copy-paste! Typing helps you learn). 以下のコードをindex.htmlファイルに入力する（コピーペーストではなく自分で入力しましょう！入力することで学べます）
  • Save the file (Ctrl+S / Cmd+S) and open it in your browser using Live Server or by double-clicking the file. ファイルを保存してブラウザで開く
  • Open Developer Tools (F12) and explore the Elements tab. Try clicking on different elements to see them highlighted. 開発者ツール（F12）を開き、要素タブを探索する

Code to Type / 入力するコード

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>My name is [Your Name]. I am a student at
    Kwansei Gakuin University.</p>

    <h2>About Me</h2>
    <p>I am studying <strong>Web Programming</strong>
    and I am excited to learn how to build websites!</p>

    <h2>My Hobbies</h2>
    <ul>
        <li>Reading</li>
        <li>Playing games</li>
        <li>Listening to music</li>
    </ul>

    <h2>My Favorite Websites</h2>
    <ol>
        <li><a href="https://www.google.com" target="_blank">Google</a></li>
        <li><a href="https://www.youtube.com" target="_blank">YouTube</a></li>
        <li><a href="https://www.kwansei.ac.jp" target="_blank">Kwansei Gakuin University</a></li>
    </ol>

    <h2>A Beautiful Place</h2>
    <img src="https://upload.wikimedia.org/wikipedia/commons/thumb/1/1b/FujiSunrise.jpg/800px-FujiSunrise.jpg"
         alt="Sunrise over Mount Fuji">
    <p><em>Mount Fuji at sunrise</em></p>

    <hr>
    <p>Thank you for visiting my page!</p>
</body>
</html>

After you finish / 終わったら: Try modifying the page! Change the text, add more list items, add another heading, or find a different image URL. The best way to learn is to experiment.

ページを修正してみましょう！テキストを変更したり、リスト項目を追加したり、別の見出しを追加したり、別の画像URLを見つけたりしてみてください。学ぶ最良の方法は実験することです。

11 Key Takeaways / 重要なポイント

1
The Web is a conversation between client and server. Your browser (client) sends a request; the server sends back HTML.
Webはクライアントとサーバーの会話です。ブラウザ（クライアント）がリクエストを送り、サーバーがHTMLを返します。
2
HTML is the skeleton of every web page. It provides structure and content, but not style or interactivity.
HTMLはすべてのWebページの骨格です。構造と内容を提供しますが、スタイルやインタラクティブ性は提供しません。
3
Every HTML page has the same basic structure: <!DOCTYPE html>, <html>, <head>, and <body>.
すべてのHTMLページは同じ基本構造を持ちます。
4
Tags come in pairs (opening and closing), except self-closing tags like <br> and <img>.
タグは（開始タグと終了タグの）ペアで使います。ただし<br>や<img>のような自己閉鎖タグは例外です。
5
Essential tags to remember: <h1>-<h6> (headings), <p> (paragraphs), <a> (links), <img> (images), <ul>/<ol>/<li> (lists).
覚えるべき基本タグ：見出し、段落、リンク、画像、リスト。
6
VS Code + Live Server is your development environment. Use it for all your coding work in this course.
VS Code + Live Serverがあなたの開発環境です。このコースのすべてのコーディング作業に使いましょう。
7
Browser Developer Tools (F12) let you inspect, debug, and experiment with any web page.
ブラウザ開発者ツール（F12）を使えば、どんなWebページでも検証、デバッグ、実験ができます。
