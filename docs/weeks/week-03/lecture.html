3. What is CSS? / CSSとは何か？

The Full Name / 正式名称

CSS stands for Cascading Style Sheets.
CSSは Cascading Style Sheets（カスケーディング・スタイル・シート）の略です。

  • Cascading = styles flow downward like a waterfall, with later/more-specific rules overriding earlier ones.
    カスケーディング = スタイルは滝のように上から下へ流れ、後のルールが前のルールを上書きします。
  • Style = the visual appearance (colors, fonts, spacing, layout).
    スタイル = 視覚的な外観（色、フォント、余白、レイアウト）。
  • Sheets = files (documents) that contain style rules.
    シート = スタイルルールを含むファイル（文書）。

A Brief History / 簡単な歴史

CSS was first proposed in 1994 by Hakon Wium Lie and was officially released as CSS1 in December 1996. Before CSS, developers had to mix appearance instructions directly into their HTML using tags like <font> and <center>. This made code messy and hard to maintain. CSS solved this by separating content (HTML) from presentation (CSS).

CSSは1994年にHakon Wium Lieによって提案され、1996年12月にCSS1として正式にリリースされました。 CSS以前は、開発者は<font>や<center>タグを使って HTML内に直接見た目の指示を書く必要がありました。 CSSは「コンテンツ（HTML）とプレゼンテーション（CSS）の分離」によってこの問題を解決しました。

The Human Body Analogy / 人体のアナロジー

Think of a website like a human body:

ウェブサイトを人体に例えて考えてみましょう：

  • HTML = Skeleton (骨格) -- The structure and bones. It holds everything in place.
    HTML = 骨格 -- 構造と骨。すべてを所定の位置に保持します。
  • CSS = Skin & Clothes (肌と服) -- The visual appearance. It determines what you see: colors, shapes, sizes, position.
    CSS = 肌と服 -- 視覚的な外観。目に見えるもの（色、形、サイズ、位置）を決定します。
  • JavaScript = Muscles & Brain (筋肉と脳) -- The behavior. It makes things move and respond. (We'll learn this starting Session 5!)
    JavaScript = 筋肉と脳 -- 動作。物事を動かし、反応させます。（セッション5から学びます！）

Same HTML, Different CSS / 同じHTML、異なるCSS

One of the most powerful ideas in web design: the same HTML content can look completely different just by changing the CSS file. This is why separation matters.

ウェブデザインで最も強力なアイデアの一つ：同じHTMLコンテンツでもCSSファイルを変えるだけで 全く異なる見た目にできます。これが分離の重要性です。

Imagine this simple HTML:

<h1>Welcome to My Site</h1>
<p>This is my homepage.</p>

With no CSS: plain black text, Times New Roman, white background.
With CSS A: blue heading, sans-serif font, light gray background.
With CSS B: red heading, handwriting font, dark background with white text.

CSSなし：地味な黒いテキスト、Times New Roman、白い背景。
CSS A：青い見出し、サンセリフフォント、薄いグレーの背景。
CSS B：赤い見出し、手書きフォント、暗い背景に白いテキスト。

4. Three Ways to Add CSS / CSSを追加する3つの方法

There are exactly three ways to add CSS to an HTML page. Each has its place, but one is clearly the best for real projects.

HTMLページにCSSを追加する方法は正確に3つあります。それぞれに用途がありますが、 実際のプロジェクトには1つが明らかに最適です。

Method 1: Inline CSS (インラインCSS)

Add the style attribute directly on an HTML element. The CSS rules go inside quotes as the value of style.

HTML要素に直接style属性を追加します。CSSルールはstyleの値として引用符の中に書きます。 

<!-- Inline CSS: style attribute on the element -->
<h1 style="color: blue; font-size: 36px;">Hello World</h1>
<p style="color: gray; font-family: Arial;">This is a paragraph.</p>

Pros / 利点: Quick and easy for testing one element.

Cons / 欠点: Not reusable! If you have 50 paragraphs, you'd need to add style="..." to each one. Very difficult to maintain.

利点：1つの要素のテスト時に素早く簡単。欠点：再利用不可！50の段落があれば、それぞれにstyle属性が必要。

Method 2: Internal CSS (内部CSS)

Add a <style> tag inside the <head> section of your HTML document. Write your CSS rules inside this tag.

HTMLドキュメントの<head>セクション内に<style>タグを追加します。 このタグの中にCSSルールを書きます。 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page</title>

    <!-- Internal CSS: style tag in the head -->
    <style>
        h1 {
            color: blue;
            font-size: 36px;
        }
        p {
            color: gray;
            font-family: Arial, sans-serif;
        }
    </style>
</head>
<body>
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Pros / 利点: Better than inline -- one rule styles ALL matching elements on that page.

Cons / 欠点: Only works on the single page where the <style> tag is. If you have 3 pages, you need to copy the same CSS into all 3 files.

利点：インラインよりも良い -- 1つのルールでページ内の全一致要素をスタイル。欠点：styleタグがあるページだけ有効。

Method 3: External CSS (外部CSS) -- THE BEST METHOD

Create a separate .css file, then link it in the <head> of your HTML using a <link> tag. This is the method used by all professional websites.

別の.cssファイルを作成し、<link>タグを使ってHTMLの<head>にリンクします。 これがすべてのプロフェッショナルなウェブサイトで使われる方法です。

Step 1: Create a file called style.css:

/* style.css -- External Stylesheet */
h1 {
    color: blue;
    font-size: 36px;
}
p {
    color: gray;
    font-family: Arial, sans-serif;
}

Step 2: Link it in your HTML file's <head>:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page</title>

    <!-- External CSS: link to a separate file -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Why External CSS Wins / 外部CSSが最も優れている理由:

  • ONE file styles ALL your pages. Change once, update everywhere.
    1つのファイルで全ページをスタイル。一度変更すれば、すべてが更新。
  • Your HTML stays clean and focused on content only.
    HTMLはきれいに保たれ、コンテンツのみに集中。
  • The browser can cache (save) the CSS file, making pages load faster.
    ブラウザがCSSファイルをキャッシュ（保存）でき、ページの読み込みが速くなる。
  • Easier for teams -- designer works on CSS, developer works on HTML.
    チーム作業が容易 -- デザイナーはCSS、開発者はHTMLを担当。

Comparison Summary / 比較まとめ

Method / 方法 Where? / 場所 Reusable? / 再利用可能？ Best For / 最適な用途
Inline style="..." on element No Quick testing only
Internal <style> in <head> Same page only Single-page projects
External <link> to .css file YES -- all pages ALL real projects

5. CSS Syntax Deep Dive / CSS構文の詳細

The Basic Rule Structure / 基本ルール構造

Every CSS rule has the same structure:

すべてのCSSルールは同じ構造を持っています：

selector {
    property: value;
    property: value;
    property: value;
}

/*
  selector  = WHAT to style (which HTML element)
  property  = WHICH aspect to change (color, size, etc.)
  value     = HOW to change it (blue, 20px, etc.)

  セレクター = 何をスタイルするか（どのHTML要素）
  プロパティ = どの側面を変えるか（色、サイズなど）
  値        = どう変えるか（青、20pxなど）
*/

Annotated Example / 注釈付きの例

/* This rule styles ALL h1 elements on the page */
/* このルールはページ内の全てのh1要素をスタイルします */
h1 {
    color: blue;            /* text color = blue / 文字色 = 青 */
    font-size: 36px;        /* text size = 36 pixels / 文字サイズ = 36ピクセル */
    font-family: Arial;     /* font = Arial / フォント = Arial */
    text-align: center;     /* center the text / テキストを中央揃え */
    margin-bottom: 20px;    /* space below = 20px / 下の余白 = 20px */
}

Key Syntax Rules / 構文の重要ルール

  • Curly braces { } wrap all properties for one selector.
    中括弧{ }は1つのセレクターの全プロパティを囲みます。
  • Colon : separates the property name from its value.
    コロン:はプロパティ名と値を区切ります。
  • Semicolon ; ends EVERY property-value pair. Do not forget this!
    セミコロン;は全てのプロパティと値のペアの末尾に付けます。忘れないでください！
  • Comments: Use /* comment here */ to add notes. The browser ignores these.
    コメント：/* コメント */でメモを追加。ブラウザはこれを無視します。
  • Case sensitivity: CSS property names and most values are case-insensitive, but class and ID names ARE case-sensitive (.MyClass is different from .myclass).
    大文字小文字：CSSプロパティ名とほとんどの値は大文字小文字を区別しませんが、 クラスとID名は区別します（.MyClassと.myclassは別物）。

Multiple Properties Example / 複数プロパティの例

/* Style a paragraph with many properties at once */
/* 段落を複数のプロパティで一度にスタイル */
p {
    color: #333333;                     /* dark gray text / 濃いグレーのテキスト */
    font-family: Arial, sans-serif;     /* font with fallback / フォント（代替あり） */
    font-size: 16px;                    /* text size / 文字サイズ */
    line-height: 1.6;                   /* line spacing / 行間 */
    margin-bottom: 15px;                /* space below / 下の余白 */
    text-align: left;                   /* align left / 左揃え */
}

6. Selectors -- Complete Guide / セレクター完全ガイド

Selectors are how you tell CSS which HTML elements to style. This is one of the most important concepts in CSS.

セレクターは、CSSにどのHTML要素をスタイルするかを指示する方法です。 これはCSSの最も重要な概念の1つです。

6a. Element Selector / 要素セレクター

Selects ALL elements of that type on the page. Just write the tag name.

ページ上のそのタイプの全要素を選択します。タグ名をそのまま書きます。

/* Style ALL h1 elements / 全てのh1要素をスタイル */
h1 {
    color: navy;
    font-size: 2rem;
}

/* Style ALL paragraphs / 全ての段落をスタイル */
p {
    color: #444;
    line-height: 1.6;
}

/* Style ALL list items / 全てのリスト項目をスタイル */
li {
    margin-bottom: 5px;
}

6b. Class Selector / クラスセレクター

Selects elements with a specific class attribute. Uses a dot . before the class name. Classes are reusable -- you can apply the same class to many elements. This is the most commonly used selector type.

特定のclass属性を持つ要素を選択します。クラス名の前にドット.を付けます。 クラスは再利用可能で、同じクラスを多くの要素に適用できます。 最もよく使われるセレクタータイプです。

In your CSS file:

/* Class selector: dot + class name */
/* クラスセレクター：ドット + クラス名 */
.highlight {
    background-color: yellow;
    padding: 5px;
}

.important-text {
    color: red;
    font-weight: bold;
}

.center-text {
    text-align: center;
}

In your HTML file:

<!-- Add class to an element with class="name" -->
<!-- class="名前" で要素にクラスを追加 -->
<p class="highlight">This text has a yellow background.</p>
<p class="important-text">This text is red and bold.</p>
<p class="center-text">This text is centered.</p>

<!-- You can use the same class on multiple elements! -->
<!-- 同じクラスを複数の要素に使えます！ -->
<h2 class="center-text">This heading is also centered.</h2>

6c. ID Selector / IDセレクター

Selects ONE specific element with a unique id attribute. Uses a hash # before the ID name. Each ID must be unique on the page -- only one element can have that ID.

一意のid属性を持つ1つの特定要素を選択します。ID名の前にハッシュ#を付けます。 各IDはページ上で一意でなければなりません。

In your CSS file:

/* ID selector: hash + id name */
/* IDセレクター：ハッシュ + ID名 */
#main-title {
    font-size: 3rem;
    color: darkblue;
    text-align: center;
}

#footer-text {
    font-size: 0.8rem;
    color: gray;
}

In your HTML file:

<!-- Add id to an element with id="name" -->
<!-- id="名前" で要素にIDを追加 -->
<h1 id="main-title">Welcome to My Website</h1>

<!-- This id can ONLY be used once per page! -->
<!-- このIDはページごとに1回しか使えません！ -->
<p id="footer-text">Copyright 2026</p>

6d. Grouping Multiple Selectors / 複数セレクターのグループ化

If you want the same styles for multiple selectors, separate them with commas.

複数のセレクターに同じスタイルを適用したい場合、カンマで区切ります。

/* Apply the same style to h1, h2, and h3 */
/* h1、h2、h3に同じスタイルを適用 */
h1, h2, h3 {
    font-family: 'Segoe UI', Arial, sans-serif;
    color: #1a1a2e;
}

/* Style both paragraphs and list items */
/* 段落とリスト項目の両方をスタイル */
p, li {
    line-height: 1.6;
    color: #333;
}

6e. Descendant Selector / 子孫セレクター

Select elements that are inside another element. Use a space between selectors.

別の要素の中にある要素を選択します。セレクター間にスペースを使います。 

/* Style only links that are INSIDE a nav element */
/* nav要素の中にあるリンクだけをスタイル */
nav a {
    color: white;
    text-decoration: none;
    padding: 10px;
}

/* Style only list items inside the footer */
/* footer内のリスト項目だけをスタイル */
footer li {
    display: inline;
    margin-right: 15px;
}

In HTML:

<nav>
    <a href="index.html">Home</a>     <!-- This link IS styled by "nav a" -->
    <a href="about.html">About</a>    <!-- This link IS styled by "nav a" -->
</nav>

<main>
    <a href="page.html">Click me</a>  <!-- This link is NOT styled by "nav a" -->
</main>

Selector Summary / セレクターまとめ

Selector Type Syntax Example Selects
Element tagname p { } All <p> elements
Class .classname .intro { } All elements with class="intro"
ID #idname #hero { } The ONE element with id="hero"
Grouping a, b, c h1, h2 { } All <h1> and <h2>
Descendant parent child nav a { } All <a> inside <nav>

7. Colors and Typography / 色とタイポグラフィ

Colors and fonts are the first things you'll want to customize. CSS gives you many properties to control how text looks.

色とフォントは最初にカスタマイズしたいものです。CSSにはテキストの見た目を制御する多くのプロパティがあります。

color -- Text Color / テキストの色

Sets the color of the text inside an element. There are three ways to specify colors:

要素内のテキストの色を設定します。色の指定方法は3つあります：

/* Named colors (about 140 available) / 色名（約140種類） */
h1 { color: red; }
h2 { color: tomato; }
h3 { color: steelblue; }

/* Hex codes: # + 6 digits (most common) / 16進コード：# + 6桁（最も一般的） */
p { color: #ff0000; }     /* red / 赤 */
p { color: #333333; }     /* dark gray / 濃いグレー */
p { color: #2563eb; }     /* blue / 青 */

/* RGB function: red, green, blue (0-255) / RGB関数 */
p { color: rgb(255, 0, 0); }     /* red / 赤 */
p { color: rgb(51, 51, 51); }    /* dark gray / 濃いグレー */

background-color -- Background Color / 背景色

body {
    background-color: #f9fafb;      /* light gray / 薄いグレー */
}

header {
    background-color: #2563eb;      /* blue / 青 */
    color: white;                   /* white text on blue background */
                                    /* 青い背景に白いテキスト */
}

.highlight {
    background-color: #fef3c7;      /* light yellow / 薄い黄色 */
}

font-family -- Font Choice / フォントの選択

Choose which font to display text in. Always provide a font stack -- a list of fonts separated by commas. The browser tries the first font; if it's not available, it tries the next one.

テキストを表示するフォントを選択します。常にフォントスタック（カンマで区切ったフォントリスト）を 提供しましょう。ブラウザは最初のフォントを試し、利用できなければ次を試します。 

/* Font stack -- list of fallback fonts / フォントスタック -- 代替フォントのリスト */
body {
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
}

/* Common web-safe font stacks / よく使われるウェブセーフフォントスタック */
h1 {
    font-family: Georgia, 'Times New Roman', Times, serif;
}

code {
    font-family: 'Courier New', Consolas, monospace;
}

/* If font name has spaces, use quotes / フォント名にスペースがある場合は引用符を使用 */
p {
    font-family: 'Segoe UI', Arial, sans-serif;
}

font-size -- Text Size / テキストサイズ

/* px = pixels (absolute, exact size) / ピクセル（絶対的、正確なサイズ） */
h1 { font-size: 36px; }
p  { font-size: 16px; }

/* em = relative to parent element's font size */
/* em = 親要素のフォントサイズに対する相対値 */
h2 { font-size: 1.5em; }   /* 1.5 times parent size / 親サイズの1.5倍 */

/* rem = relative to root (html) element's font size (RECOMMENDED) */
/* rem = ルート（html）要素のフォントサイズに対する相対値（推奨） */
h1 { font-size: 2.5rem; }  /* 2.5 times root size (usually 40px) */
p  { font-size: 1rem; }    /* same as root size (usually 16px) */

Tip / ヒント: Use rem for font sizes in your projects. It is consistent and scales well.
プロジェクトではフォントサイズにremを使いましょう。一貫性があり、スケールも良好です。

font-weight -- Bold / 太さ

/* Keyword values / キーワード値 */
p { font-weight: normal; }     /* regular weight (400) / 通常の太さ */
strong { font-weight: bold; }  /* bold weight (700) / 太字 */

/* Numeric values: 100 to 900 / 数値：100から900 */
h1 { font-weight: 900; }  /* extra bold / 極太 */
h2 { font-weight: 700; }  /* bold / 太字 */
h3 { font-weight: 500; }  /* medium / ミディアム */
p  { font-weight: 300; }  /* light / 細字 */

text-align -- Horizontal Alignment / 水平揃え

h1 { text-align: center; }   /* centered / 中央揃え */
p  { text-align: left; }     /* left-aligned (default) / 左揃え（デフォルト） */
.price { text-align: right; } /* right-aligned / 右揃え */

text-decoration -- Underline, etc. / 下線など

/* Remove underline from links (very common!) */
/* リンクから下線を削除（非常によく使う！） */
a {
    text-decoration: none;
}

/* Add underline on hover / ホバー時に下線を追加 */
a:hover {
    text-decoration: underline;
}

/* Other values / その他の値 */
.strikethrough { text-decoration: line-through; }  /* 取り消し線 */
.underlined    { text-decoration: underline; }     /* 下線 */

line-height -- Line Spacing / 行間

/* Unitless number = multiplier of font-size (RECOMMENDED) */
/* 単位なしの数値 = フォントサイズの倍数（推奨） */
p {
    font-size: 16px;
    line-height: 1.6;   /* 16px * 1.6 = 25.6px between lines / 行間25.6px */
}

body {
    line-height: 1.6;   /* Good default for readability / 読みやすさの良いデフォルト */
}

8. The Box Model -- In Depth / ボックスモデル詳細

Every Element is a Box / すべての要素はボックス

This is one of the most fundamental concepts in CSS: every HTML element is a rectangular box, even if it doesn't look like one. Understanding the Box Model is essential for controlling spacing and layout.

CSSの最も基本的な概念の1つ：すべてのHTML要素は長方形のボックスです。 ボックスモデルの理解は、余白とレイアウトの制御に不可欠です。

The Four Layers / 4つの層

Every box has four layers, from inside to outside:

すべてのボックスには、内側から外側へ4つの層があります：

+--------------------------------------------------+ | MARGIN (外側余白) | | (transparent space outside the border) | | | | +--------------------------------------------+ | | | BORDER (枠線) | | | | (visible line around the element) | | | | | | | | +--------------------------------------+ | | | | | PADDING (内側余白) | | | | | | (space inside the border) | | | | | | | | | | | | +------------------------------+ | | | | | | | CONTENT (コンテンツ) | | | | | | | | (your text, images, etc.) | | | | | | | | width and height apply | | | | | | | | here by default | | | | | | | +------------------------------+ | | | | | | | | | | | +--------------------------------------+ | | | | | | | +--------------------------------------------+ | | | +--------------------------------------------------+

Layer by Layer / 各層の詳細

1. Content (コンテンツ):

The actual text, images, or other elements inside. The width and height properties set the size of this area (by default). 

.box {
    width: 300px;     /* content area is 300px wide / コンテンツ領域の幅300px */
    height: 200px;    /* content area is 200px tall / コンテンツ領域の高さ200px */
}

2. Padding (パディング / 内側余白):

Space between the content and the border. Padding pushes the content inward, creating breathing room inside the element. Padding has the element's background color.

コンテンツとボーダーの間のスペース。パディングはコンテンツを内側に押し、 要素内に余裕を作ります。パディングには要素の背景色が適用されます。 

.box {
    padding: 20px;                    /* 20px on ALL four sides / 全4辺に20px */
    padding: 10px 20px;               /* 10px top/bottom, 20px left/right */
                                      /* 上下10px、左右20px */
    padding: 10px 20px 30px 40px;     /* top right bottom left (clockwise) */
                                      /* 上 右 下 左（時計回り） */
}

/* You can also set individual sides / 個別の辺も設定可能 */
.box {
    padding-top: 10px;
    padding-right: 20px;
    padding-bottom: 30px;
    padding-left: 40px;
}

3. Border (ボーダー / 枠線):

A visible line around the padding. You specify width, style, and color.

パディングの周りに見える線。幅、スタイル、色を指定します。

.box {
    border: 1px solid black;         /* thin solid black line / 細い実線の黒い枠 */
    border: 2px dashed #2563eb;      /* dashed blue line / 青い破線 */
    border: 3px dotted red;          /* dotted red line / 赤い点線 */
}

/* Border on specific sides / 特定の辺のボーダー */
.box {
    border-bottom: 2px solid #2563eb;  /* only bottom border / 下のボーダーのみ */
    border-left: 4px solid #7c3aed;    /* only left border / 左のボーダーのみ */
}

/* Rounded corners / 角丸 */
.box {
    border-radius: 10px;              /* rounded corners / 角丸 */
    border-radius: 50%;               /* makes a circle (if square) / 正方形なら円になる */
}

4. Margin (マージン / 外側余白):

Space outside the border. Margin creates distance between this element and neighboring elements. Margin is always transparent.

ボーダーの外側のスペース。マージンはこの要素と隣接する要素の間に距離を作ります。 マージンは常に透明です。 

.box {
    margin: 20px;                     /* 20px on ALL sides / 全辺に20px */
    margin: 10px 20px;                /* 10px top/bottom, 20px left/right */
    margin: 10px 20px 30px 40px;      /* top right bottom left (clockwise) */
}

/* Center an element horizontally! / 要素を水平方向に中央揃え！ */
.centered-box {
    width: 600px;
    margin: 0 auto;    /* 0 top/bottom, auto left/right = centered! */
                       /* 上下0、左右auto = 中央揃え！ */
}

The Problem: Default Box Sizing / デフォルトのボックスサイズの問題

By default, width and height only set the content area. Padding and border are added to the total size. This is confusing:

デフォルトでは、widthとheightはコンテンツ領域のみを設定します。 パディングとボーダーは合計サイズに追加されます。これは紛らわしいです： 

/* Without border-box / border-boxなし */
.box {
    width: 300px;
    padding: 20px;
    border: 5px solid black;
}
/* Total width = 300 + 20 + 20 + 5 + 5 = 350px!  Surprising! */
/* 合計幅 = 300 + 20 + 20 + 5 + 5 = 350px！ 驚き！ */

The Solution: box-sizing: border-box

This magical property makes width include padding AND border. The content area shrinks to fit. This is much more intuitive.

この魔法のプロパティにより、widthにパディングとボーダーが含まれます。 コンテンツ領域はフィットするように縮小されます。これはずっと直感的です。 

/* Apply to ALL elements (best practice) / 全要素に適用（ベストプラクティス） */
* {
    box-sizing: border-box;
}

/* Now width = TOTAL width including padding and border */
/* これでwidth = パディングとボーダーを含む合計幅 */
.box {
    width: 300px;
    padding: 20px;
    border: 5px solid black;
}
/* Total width = 300px exactly! Content area adjusts automatically. */
/* 合計幅 = 正確に300px！コンテンツ領域が自動的に調整。 */

Best Practice / ベストプラクティス: Always add * { box-sizing: border-box; } at the top of your CSS file. Most professional websites do this.
CSSファイルの先頭に必ず* { box-sizing: border-box; }を追加しましょう。

9. Introduction to Flexbox / Flexbox入門

The Problem / 問題

By default, HTML block elements stack vertically -- each element appears below the previous one. But what if you want elements side by side, like a navigation bar? Before Flexbox, this was surprisingly difficult.

デフォルトでは、HTMLのブロック要素は垂直方向に積み重なります。 ナビゲーションバーのように要素を横並びにしたい場合はどうするか？Flexbox以前は驚くほど困難でした。

The Solution: display: flex

Add display: flex to a parent container and its child elements will automatically line up in a row.

親コンテナにdisplay: flexを追加すると、子要素が自動的に横一列に並びます。 

/* The container becomes a flex container / コンテナがフレックスコンテナになる */
.flex-container {
    display: flex;   /* This one line changes everything! */
                     /* この1行ですべてが変わる！ */
}

HTML:

<div class="flex-container">
    <div>Item 1</div>
    <div>Item 2</div>
    <div>Item 3</div>
</div>

<!-- Without flex: items stack vertically (top to bottom) -->
<!-- With flex: items line up horizontally (left to right) -->

flex-direction -- Direction / 方向

.container {
    display: flex;
    flex-direction: row;      /* left to right (DEFAULT) / 左から右（デフォルト） */
    flex-direction: column;   /* top to bottom (like normal) / 上から下（通常通り） */
}

justify-content -- Horizontal Alignment / 水平方向の配置

Controls how items are positioned along the main axis (horizontal by default).

主軸（デフォルトでは水平方向）に沿ったアイテムの配置を制御します。

.container {
    display: flex;
    justify-content: flex-start;      /* pack items to the left (default) */
                                      /* アイテムを左に詰める（デフォルト） */
    justify-content: center;          /* center items horizontally */
                                      /* アイテムを水平方向に中央揃え */
    justify-content: flex-end;        /* pack items to the right */
                                      /* アイテムを右に詰める */
    justify-content: space-between;   /* equal space BETWEEN items */
                                      /* アイテム間に均等なスペース */
    justify-content: space-around;    /* equal space AROUND each item */
                                      /* 各アイテムの周りに均等なスペース */
}

align-items -- Vertical Alignment / 垂直方向の配置

Controls how items are positioned along the cross axis (vertical by default).

交差軸（デフォルトでは垂直方向）に沿ったアイテムの配置を制御します。

.container {
    display: flex;
    align-items: stretch;     /* items stretch to fill height (default) */
                              /* アイテムが高さいっぱいに伸びる（デフォルト） */
    align-items: center;      /* center items vertically */
                              /* アイテムを垂直方向に中央揃え */
    align-items: flex-start;  /* align items to the top */
                              /* アイテムを上に揃える */
    align-items: flex-end;    /* align items to the bottom */
                              /* アイテムを下に揃える */
}

gap -- Spacing Between Items / アイテム間の間隔

.container {
    display: flex;
    gap: 20px;     /* 20px space between each item / 各アイテム間に20pxのスペース */
}

Practical Example: Navigation Bar / 実践例：ナビゲーションバー

Let's build a horizontal navigation bar using Flexbox -- one of the most common uses.

Flexboxを使って水平ナビゲーションバーを作りましょう -- 最も一般的な使い方の1つです。

HTML:

<nav class="navbar">
    <a href="index.html">Home</a>
    <a href="about.html">About</a>
    <a href="hobbies.html">Hobbies</a>
</nav>

CSS:

.navbar {
    display: flex;              /* make children horizontal / 子要素を横並びに */
    gap: 20px;                  /* space between links / リンク間のスペース */
    background-color: #1f2937;  /* dark background / 暗い背景 */
    padding: 15px 20px;         /* inner spacing / 内側の余白 */
}

.navbar a {
    color: white;               /* white text / 白いテキスト */
    text-decoration: none;      /* remove underline / 下線を削除 */
    padding: 8px 16px;          /* clickable area / クリック可能な領域 */
    border-radius: 5px;         /* rounded corners / 角丸 */
}

.navbar a:hover {
    background-color: #2563eb;  /* blue on hover / ホバー時に青 */
}

この<link>タグはすべてのHTMLファイルに追加する必要があります。 1ページでも忘れると、そのページにスタイルが適用されません！

The Cascade -- How Styles Override / カスケード -- スタイルの上書き

When multiple styles apply to the same element, CSS follows a priority order (lowest to highest):

同じ要素に複数のスタイルが適用される場合、CSSは優先順位に従います（低から高）：

  1. Browser defaults -- the browser's built-in styles (e.g., blue underlined links)
    ブラウザのデフォルト -- ブラウザの組み込みスタイル（例：青い下線付きリンク）
  2. External CSS -- your .css file
    外部CSS -- あなたの.cssファイル
  3. Internal CSS -- <style> tag in the page
    内部CSS -- ページ内の<style>タグ
  4. Inline CSS -- style="..." attribute (highest priority!)
    インラインCSS -- style="..."属性（最高優先度！）

Specificity -- Which Selector Wins? / 詳細度 -- どのセレクターが勝つか？

When two CSS rules target the same element, the more specific selector wins:

2つのCSSルールが同じ要素を対象とする場合、より具体的なセレクターが勝ちます：

Selector Type Specificity Example
Element Lowest (lowest priority) p { color: black; }
Class Medium .intro { color: blue; }
ID High #main { color: red; }
Inline Highest (wins over everything) style="color: green;" 
/* If an element has class="intro" and id="main": */
/* 要素が class="intro" と id="main" を持つ場合: */

p     { color: black; }    /* specificity: LOW    -- overridden */
.intro { color: blue; }    /* specificity: MEDIUM -- overridden */
#main  { color: red; }     /* specificity: HIGH   -- THIS WINS! */

/* The text will be RED because #id beats .class beats element */
/* テキストは赤になる。#idが.classに勝ち、.classが要素に勝つから */

13. Common CSS Mistakes / よくあるCSSミス

Even experienced developers make these mistakes. Watch out for them!

経験豊富な開発者でもこれらの間違いをします。注意しましょう！

Mistake 1: Missing Semicolons / セミコロンの欠落

/* WRONG -- missing semicolon after "blue" / 間違い -- "blue"の後にセミコロンがない */
h1 {
    color: blue       /* <-- missing ; here! */
    font-size: 24px;
}
/* Result: font-size will NOT work because the browser is confused */
/* 結果: ブラウザが混乱し、font-sizeが機能しない */

/* CORRECT / 正解 */
h1 {
    color: blue;      /* <-- semicolon present! */
    font-size: 24px;
}

Mistake 2: Misspelling Property Names / プロパティ名のスペルミス

/* WRONG / 間違い */
p {
    colour: red;          /* it's "color" not "colour" (American English) */
    font-colour: blue;    /* this property does not exist at all */
    backgroud-color: red; /* typo: "backgroud" instead of "background" */
}

/* CORRECT / 正解 */
p {
    color: red;
    background-color: blue;
}

Mistake 3: Forgetting to Link the CSS File / CSSファイルのリンク忘れ

<!-- WRONG -- no link tag! Your CSS file will be ignored. -->
<!-- 間違い -- linkタグがない！CSSファイルが無視されます。 -->
<head>
    <title>My Page</title>
</head>

<!-- CORRECT -- link tag present -->
<!-- 正解 -- linkタグあり -->
<head>
    <title>My Page</title>
    <link rel="stylesheet" href="style.css">
</head>

Mistake 4: Class Names with Spaces / スペースのあるクラス名

<!-- WRONG -- this creates TWO classes: "my" and "class" -->
<!-- 間違い -- これは2つのクラスを作成："my" と "class" -->
<p class="my class">Text</p>

<!-- CORRECT -- use a hyphen or camelCase for multi-word class names -->
<!-- 正解 -- 複数語のクラス名にはハイフンまたはキャメルケースを使用 -->
<p class="my-class">Text</p>
<p class="myClass">Text</p>

<!-- If you WANT two classes, that is valid, but intentional: -->
<!-- 2つのクラスを意図的に使いたい場合は有効： -->
<p class="bold red">This has BOTH the "bold" and "red" classes.</p>

Debugging Tip / デバッグのヒント: Use your browser's Developer Tools (press F12 or right-click and choose "Inspect") to see which CSS rules are being applied and which are being overridden!

ブラウザの開発者ツール（F12を押すか、右クリックで「検証」を選択）を使って、 どのCSSルールが適用され、どれが上書きされているかを確認しましょう！

14. Key Takeaways / 重要ポイント

  • CSS separates style from content -- HTML defines structure, CSS defines appearance.
    CSSはスタイルをコンテンツから分離する -- HTMLは構造、CSSは外観を定義。
  • Always use external CSS (<link> to a .css file) for real projects. One file, all pages.
    実プロジェクトでは常に外部CSSを使用。1つのファイルで全ページ。
  • Selectors target elements: element (p), class (.name), ID (#name), descendants (nav a).
    セレクターは要素を特定：要素、クラス、ID、子孫。
  • The Box Model has 4 layers: content, padding, border, margin. Always use box-sizing: border-box.
    ボックスモデルは4層：コンテンツ、パディング、ボーダー、マージン。常にborder-boxを使用。
  • Flexbox makes horizontal layout easy: display: flex on the parent, use gap, justify-content, and align-items.
    Flexboxは横並びレイアウトを簡単に：親にdisplay: flex、gap、justify-content、align-itemsを使用。
  • Specificity determines which style wins: element < class < ID < inline.

  • Use Developer Tools (F12) to inspect and debug your CSS.
