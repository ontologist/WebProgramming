3 What is JavaScript? / JavaScriptとは？

JavaScript is a programming language -- the most popular programming language in the world. Unlike HTML (which describes structure) and CSS (which describes appearance), JavaScript can do things: perform calculations, make decisions, respond to user actions, and draw graphics.

JavaScriptはプログラミング言語であり、世界で最も人気のあるプログラミング言語です。HTML（構造を記述）やCSS（見た目を記述）とは異なり、JavaScriptは計算をしたり、判断をしたり、ユーザーの操作に反応したり、グラフィックスを描いたりすることができます。

A Brief History / 簡単な歴史

JavaScript was created by Brendan Eich in 1995 at Netscape Communications. He famously built the first version in just 10 days! Despite the name, JavaScript has nothing to do with Java -- the similar name was a marketing decision. Today, JavaScript runs in every web browser on every device in the world.

JavaScriptは1995年にNetscape Communicationsのブレンダン・アイクによって作られました。最初のバージョンはわずか10日で作られたと言われています！名前は似ていますが、JavaScriptはJavaとは無関係です -- 名前が似ているのはマーケティング上の判断でした。現在、JavaScriptは世界中のすべてのデバイスのすべてのWebブラウザで動作しています。

Why JavaScript Matters / JavaScriptが重要な理由

  • Interactivity: JavaScript makes web pages respond to user actions (clicks, keyboard input, mouse movement). / ユーザーの操作に反応するページを作れる
  • Dynamic content: JavaScript can change what is displayed on the page without reloading it. / ページを再読み込みせずに表示内容を変更できる
  • Graphics and animation: JavaScript can draw shapes, create animations, and build games (which is exactly what we will do!). / 図形の描画、アニメーション、ゲーム制作ができる
  • Universal: Every web browser can run JavaScript. No installation needed. / すべてのWebブラウザで動作する。インストール不要

Important distinction / 重要な区別: HTML and CSS are not programming languages. They are markup and styling languages respectively. JavaScript IS a programming language because it can perform logic, make decisions, and execute sequences of instructions.

HTMLとCSSはプログラミング言語ではありません。それぞれマークアップ言語とスタイリング言語です。JavaScriptはロジックを実行し、判断を下し、命令のシーケンスを実行できるため、プログラミング言語です。

Analogy / たとえ話: If a web page is a restaurant:

  • HTML = the building, tables, and chairs (structure) / 建物、テーブル、椅子（構造）
  • CSS = the paint, decorations, and tablecloths (appearance) / ペンキ、装飾、テーブルクロス（見た目）
  • JavaScript = the waiters, chefs, and kitchen (behavior -- things that actually DO something) / ウェイター、シェフ、厨房（動作 -- 実際に何かをするもの）

4 The <script> Tag / <script>タグ

To add JavaScript to your HTML page, you use the <script> tag. There are two ways to include JavaScript:

HTMLページにJavaScriptを追加するには、<script>タグを使います。JavaScriptを含める方法は2つあります：

Method 1: Inline Script (What We Will Use) / 方法1：インラインスクリプト（今回使う方法）

Write your JavaScript code directly inside the <script> tag:

<!DOCTYPE html>
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <h1>Hello!</h1>

    <script>
        // JavaScript code goes here
        // Lines starting with // are comments (the browser ignores them)
    </script>
</body>
</html>

Method 2: External Script (For Later) / 方法2：外部スクリプト（後で学ぶ）

You can also put your JavaScript in a separate .js file and link to it. We will learn this method later in the course.

<script src="game.js"></script>

Where to Place the <script> Tag / <script>タグの配置場所

Always place your <script> tag just before </body> (the closing body tag).

<script>タグは常に</body>（bodyの閉じタグ）の直前に置きましょう。

Why? The browser reads HTML from top to bottom. If your script tries to access an HTML element that hasn't been loaded yet, it will fail. By placing the script at the bottom, all HTML elements are already available.

なぜ？ブラウザはHTMLを上から下に読みます。スクリプトがまだ読み込まれていないHTML要素にアクセスしようとすると失敗します。スクリプトを下に置けば、すべてのHTML要素がすでに利用可能です。

Comments in JavaScript / JavaScriptのコメント

Comments are notes in your code that the browser ignores. They help you (and others) understand what your code does.

// This is a single-line comment (starts with //)

/*
  This is a multi-line comment.
  It can span several lines.
  Very useful for longer explanations.
*/

Best practice / ベストプラクティス: Write comments in your code to explain why you are doing something, not just what you are doing. For example: // Set ball color to blue to match team colors is better than // Set color to blue.

コードにコメントを書いて、何をしているかだけでなく、なぜそうしているかを説明しましょう。

5 Variables: let and const / 変数：letとconst

A variable is a named container that stores a value. Think of it like a labeled box: you write a name on the box and put something inside it.

変数は値を保存する名前付きの入れ物です。ラベル付きの箱のように考えてください：箱に名前を書いて、中に何かを入れます。

Declaring Variables with let / letで変数を宣言する

Use let when you want to create a variable whose value might change later:

let score = 0;           // score starts at 0
score = 10;              // later, score changes to 10 (this is OK!)

let playerName = "Yuki"; // a text variable
let ballX = 240;         // a number variable

Declaring Constants with const / constで定数を宣言する

Use const when you want to create a variable whose value should never change:

const canvasWidth = 480;   // the canvas is always 480 pixels wide
const canvasHeight = 320;  // the canvas is always 320 pixels tall
const ballColor = "red";   // the ball color does not change

// canvasWidth = 500;  // ERROR! You cannot change a const!

Analogy / たとえ話:

  • let is like a whiteboard -- you can erase and rewrite the value. / letはホワイトボードのようなもの -- 値を消して書き直せる
  • const is like a stone engraving -- once set, it cannot be changed. / constは石の彫刻のようなもの -- 一度設定したら変更できない

Variable Naming Rules / 変数の命名ルール

Rule / ルール Good Example Bad Example
Must start with a letter, _, or $ / 文字、_、$で始める ballX, _score 2balls, -name
No spaces allowed / スペースは使えない ballColor ball color
Case-sensitive / 大文字と小文字を区別する ballX and BallX are different --
Use camelCase for multi-word names / 複数語にはキャメルケースを使う paddleWidth, ballRadius paddlewidth, paddle_width
Use meaningful names / 意味のある名前を付ける ballX, score x1, a

Tip / ヒント: When in doubt, use const. Only use let when you know the value needs to change. This helps prevent accidental changes to important values.

迷ったらconstを使いましょう。値を変更する必要があるとわかっている場合にのみletを使います。これにより、重要な値の偶発的な変更を防げます。

6 Data Types / データ型

Every value in JavaScript has a type. For today, we focus on the two most important types:

JavaScriptのすべての値には型があります。今日は最も重要な2つの型に集中します：

Numbers (数値) / 数値型

Numbers are used for positions, sizes, scores, and any kind of math:

let x = 100;          // integer (whole number)
let y = 3.14;         // floating point (decimal)
let width = 480;
let radius = 10;

// Arithmetic operators (math)
let sum = 10 + 5;     // 15  (addition)
let diff = 10 - 5;    // 5   (subtraction)
let product = 10 * 5; // 50  (multiplication)
let quotient = 10 / 5;// 2   (division)

Strings (文字列) / 文字列型

Strings are text values. They must be wrapped in quotes (single or double):

let color = "red";           // double quotes
let name = 'Canvas Drawing'; // single quotes -- both work!
let hex = "#0095DD";         // hex color codes are strings too

// String concatenation (joining strings together)
let greeting = "Hello, " + "World!";  // "Hello, World!"

Common mistake / よくある間違い: Do not confuse numbers and strings!

let a = 5 + 3; results in 8 (number addition).

let b = "5" + "3"; results in "53" (string concatenation, joining text).

数値と文字列を混同しないでください！5 + 3は8（数値の足し算）、"5" + "3"は"53"（文字列の結合）になります。

7 The <canvas> Element / <canvas>要素

The <canvas> element is an HTML tag that creates a rectangular drawing area on your page. By itself, a canvas is invisible -- it is just an empty space. You need JavaScript to draw on it.

<canvas>要素は、ページ上に長方形の描画領域を作成するHTMLタグです。canvas自体は目に見えません -- ただの空の空間です。描画にはJavaScriptが必要です。

Creating a Canvas / Canvasの作成

<canvas id="myCanvas" width="480" height="320"></canvas>
Attribute Purpose / 目的 Example
id A unique name so JavaScript can find this element / JavaScriptがこの要素を見つけるための一意の名前 id="myCanvas"
width The width of the drawing area in pixels / 描画領域の幅（ピクセル） width="480"
height The height of the drawing area in pixels / 描画領域の高さ（ピクセル） height="320"

Tip / ヒント: Add a border so you can see the canvas: <canvas id="myCanvas" width="480" height="320" style="border: 1px solid black;"></canvas>

canvasを見えるようにするためにボーダーを追加しましょう。

Getting the 2D Context / 2Dコンテキストの取得

Before you can draw anything, you need to get the 2D rendering context -- the object that provides all the drawing methods:

何かを描く前に、2D描画コンテキストを取得する必要があります -- これがすべての描画メソッドを提供するオブジェクトです：

// Step 1: Find the canvas element in the HTML
const canvas = document.getElementById("myCanvas");

// Step 2: Get the 2D drawing context
const ctx = canvas.getContext("2d");

// Now ctx has all the drawing methods!

Let's break this down:

  • document refers to the entire web page. / Webページ全体を指す
  • .getElementById("myCanvas") searches the page for an element with id="myCanvas". / id="myCanvas"の要素をページから検索する
  • .getContext("2d") returns the 2D drawing tools. We store it in ctx (short for "context"). / 2D描画ツールを返す。"context"の略でctxに保存する

Analogy / たとえ話: Think of the canvas as a blank sheet of paper on a desk. document.getElementById is like reaching over and picking up that specific sheet. getContext("2d") is like picking up a set of colored pencils -- now you have the tools to draw.

canvasは机の上の白い紙だと考えてください。document.getElementByIdはその特定の紙を手に取ること。getContext("2d")は色鉛筆のセットを手に取ること -- これで描画するツールが手に入りました。

8 The Coordinate System / 座標系

The canvas uses an (x, y) coordinate system to specify where to draw things. However, it works differently from the coordinate system you learned in math class:

Canvasは、描画位置を指定するために(x, y)座標系を使います。ただし、数学の授業で学んだ座標系とは異なる動きをします：

Canvas Coordinate System / Canvas座標系
  (0,0)───────────────────────> x (right / 右)
    │
    │    (100, 50) means:
    │    100 pixels from left
    │    50 pixels from top
    │
    │         * (100, 50)
    │
    │
    │
    ▼
  y (down / 下)

  Canvas size: 480 x 320 pixels
  Bottom-right corner = (480, 320)

Key Points / 要点

  • (0, 0) is at the top-left corner, not the center or bottom-left. / (0, 0)は中央や左下ではなく、左上の角にある
  • x increases as you move to the right. / xは右に移動するほど増える
  • y increases as you move down. / yは下に移動するほど増える
  • All measurements are in pixels. / すべての測定値はピクセル単位

The biggest gotcha for beginners / 初心者にとって最大の落とし穴: In math class, y goes UP. On a canvas, y goes DOWN. So a y value of 0 is the TOP of the canvas, and a y value of 320 is the BOTTOM. This is because computer screens draw from the top-left corner, one row of pixels at a time, going downward.

数学の授業ではyは上に向かいますが、canvasではyは下に向かいます。yが0はcanvasの上端で、yが320は下端です。これはコンピュータの画面が左上の角から1行ずつ下に向かってピクセルを描画するためです。

Examples / 例

Coordinate / 座標 Location / 位置
(0, 0)Top-left corner / 左上の角
(480, 0)Top-right corner / 右上の角
(0, 320)Bottom-left corner / 左下の角
(480, 320)Bottom-right corner / 右下の角
(240, 160)Center of the canvas / canvasの中央

9 Drawing Rectangles / 長方形の描画

Rectangles are the simplest shapes to draw on canvas. There are three rectangle methods:

長方形はcanvas上で最も簡単に描ける図形です。3つの長方形メソッドがあります：

fillRect(x, y, width, height) -- Filled Rectangle / 塗りつぶし長方形

Draws a solid, filled rectangle:

// Draw a filled rectangle at position (20, 40), size 150x100
ctx.fillStyle = "blue";        // set color first!
ctx.fillRect(20, 40, 150, 100);

The four parameters are:

  • x = the x-coordinate of the rectangle's top-left corner / 長方形の左上のx座標
  • y = the y-coordinate of the rectangle's top-left corner / 長方形の左上のy座標
  • width = how wide the rectangle is (in pixels) / 長方形の幅（ピクセル）
  • height = how tall the rectangle is (in pixels) / 長方形の高さ（ピクセル）

strokeRect(x, y, width, height) -- Outlined Rectangle / 枠線のみの長方形

Draws only the outline (border) of a rectangle:

ctx.strokeStyle = "red";          // outline color
ctx.lineWidth = 3;                // outline thickness
ctx.strokeRect(20, 40, 150, 100); // just the border, not filled

clearRect(x, y, width, height) -- Erase a Rectangle / 長方形の消去

Erases everything inside the specified rectangle area, making it transparent:

// Erase the entire canvas
ctx.clearRect(0, 0, canvas.width, canvas.height);

Note / 注意: clearRect will be very important later when we learn animation. To animate, you erase the entire canvas and redraw everything at a new position.

clearRectはアニメーションを学ぶときに非常に重要になります。アニメーションでは、canvas全体を消去して新しい位置にすべてを再描画します。

Practical Example: Drawing a Paddle / 実践例：パドルの描画

// Paddle properties stored in variables
const paddleWidth = 100;
const paddleHeight = 15;
const paddleX = 190;       // centered on a 480px canvas: (480 - 100) / 2
const paddleY = 300;       // near the bottom of a 320px canvas
const paddleColor = "#2563eb";

// Draw the paddle
ctx.fillStyle = paddleColor;
ctx.fillRect(paddleX, paddleY, paddleWidth, paddleHeight);

10 Drawing Circles and Arcs / 円と弧の描画

Unlike rectangles, there is no single "drawCircle" method. Instead, we use the arc() method inside a path. A path is like a series of pen strokes that define a shape.

長方形とは異なり、「drawCircle」のような単一のメソッドはありません。代わりに、パスの中でarc()メソッドを使います。パスは形状を定義する一連のペンストロークのようなものです。

The arc() Method / arc()メソッド

ctx.arc(centerX, centerY, radius, startAngle, endAngle);
Parameter / パラメータ Meaning / 意味
centerXThe x-coordinate of the circle's center / 円の中心のx座標
centerYThe y-coordinate of the circle's center / 円の中心のy座標
radiusThe radius (half the diameter) of the circle / 円の半径（直径の半分）
startAngleWhere to start drawing (use 0 for a full circle) / 描画の開始角度（完全な円は0）
endAngleWhere to stop drawing (use Math.PI * 2 for a full circle) / 描画の終了角度（完全な円はMath.PI * 2）

Why Math.PI * 2? / なぜMath.PI * 2？ A full circle is 360 degrees, which equals 2 times Pi radians. JavaScript uses radians, not degrees. Math.PI is a built-in constant that equals approximately 3.14159. So Math.PI * 2 is approximately 6.28, which is a full circle.

完全な円は360度で、これは2パイ・ラジアンに等しいです。JavaScriptは度ではなくラジアンを使います。Math.PIは約3.14159の組み込み定数です。

The Path Pattern / パスのパターン

When drawing circles (and other non-rectangular shapes), you must follow this pattern:

// 1. Start a new path
ctx.beginPath();

// 2. Define the shape
ctx.arc(240, 160, 10, 0, Math.PI * 2);

// 3. Set the color and fill (or stroke for outline only)
ctx.fillStyle = "#0095DD";
ctx.fill();

// 4. Close the path
ctx.closePath();

Here is what each step does:

  • beginPath() -- Tells the canvas: "I am starting a new shape." This is like picking up a pen. / 「新しい形を描き始める」とcanvasに伝える。ペンを持ち上げるようなもの
  • arc() -- Defines the shape (a circle in this case). This is like moving the pen in a circle. / 形を定義する（この場合は円）。ペンを円形に動かすようなもの
  • fillStyle and fill() -- Sets the color and fills the shape with that color. / 色を設定し、その色で形を塗りつぶす
  • closePath() -- Tells the canvas: "I am done with this shape." This is like putting the pen down. / 「この形の描画を終了する」とcanvasに伝える。ペンを置くようなもの

Do not forget beginPath()! / beginPath()を忘れないで！ If you draw multiple circles without calling beginPath() each time, they will all be connected together as one shape, and the color of the last one will apply to all of them.

毎回beginPath()を呼ばずに複数の円を描くと、すべてが1つの形として接続され、最後の色がすべてに適用されます。

Practical Example: Drawing a Ball / 実践例：ボールの描画

// Ball properties stored in variables
const ballX = 240;           // center of a 480px canvas
const ballY = 160;           // center of a 320px canvas
const ballRadius = 10;
const ballColor = "#0095DD";

// Draw the ball
ctx.beginPath();
ctx.arc(ballX, ballY, ballRadius, 0, Math.PI * 2);
ctx.fillStyle = ballColor;
ctx.fill();
ctx.closePath();

Drawing Just an Outline Circle / 枠線だけの円を描く

Use stroke() instead of fill() to draw only the outline:

ctx.beginPath();
ctx.arc(240, 160, 30, 0, Math.PI * 2);
ctx.strokeStyle = "black";
ctx.lineWidth = 2;
ctx.stroke();       // outline only, not filled
ctx.closePath();

11 Colors and Styles / 色とスタイル

Before drawing any shape, you can set various style properties to control how it looks:

図形を描く前に、見た目を制御するさまざまなスタイルプロパティを設定できます：

fillStyle -- Fill Color / 塗りつぶしの色

Sets the color used to fill shapes:

// Color name
ctx.fillStyle = "red";

// Hexadecimal color code
ctx.fillStyle = "#0095DD";

// RGB color
ctx.fillStyle = "rgb(0, 149, 221)";

// RGBA color (with transparency: 0 = invisible, 1 = solid)
ctx.fillStyle = "rgba(0, 149, 221, 0.5)"; // 50% transparent

strokeStyle -- Outline Color / 枠線の色

Sets the color used for outlines (borders):

ctx.strokeStyle = "black";
ctx.strokeStyle = "#ff0000"; // red in hex

lineWidth -- Outline Thickness / 枠線の太さ

ctx.lineWidth = 1;   // thin (default)
ctx.lineWidth = 3;   // medium
ctx.lineWidth = 10;  // thick

Common Color Names / よく使う色の名前

Color Name Hex Code Japanese / 日本語
"red"#ff0000赤
"blue"#0000ff青
"green"#008000緑
"yellow"#ffff00黄
"orange"#ffa500オレンジ
"purple"#800080紫
"white"#ffffff白
"black"#000000黒

Important order / 重要な順序: Always set fillStyle or strokeStyle before calling fillRect(), fill(), strokeRect(), or stroke(). The style must be set before the drawing command, not after. Think of it like dipping your paintbrush in paint before you paint.

fillStyleやstrokeStyleは、描画コマンドの前に設定してください。絵の具に筆を浸してから塗るのと同じです。

12 Putting It All Together / すべてをまとめる

Let's combine everything we learned into a complete example. This code draws a simple scene with a ball and a paddle on a canvas -- the beginnings of our Breakout game!

学んだことをすべて組み合わせて、完全な例を作りましょう。このコードはcanvas上にボールとパドルのある簡単なシーンを描きます -- ブロック崩しゲームの始まりです！

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Canvas Drawing - Ball and Paddle</title>
</head>
<body>
    <h1>My Canvas Drawing</h1>
    <canvas id="myCanvas" width="480" height="320"
        style="border: 2px solid #333; background: #eee;"></canvas>

    <script>
        // === SETUP ===
        // Get the canvas and its 2D drawing context
        const canvas = document.getElementById("myCanvas");
        const ctx = canvas.getContext("2d");

        // === BALL VARIABLES ===
        const ballX = 240;           // horizontal center
        const ballY = 160;           // vertical center
        const ballRadius = 12;       // size of the ball
        const ballColor = "#0095DD"; // blue color

        // === PADDLE VARIABLES ===
        const paddleWidth = 100;     // width of the paddle
        const paddleHeight = 15;     // height of the paddle
        const paddleX = 190;         // (480 - 100) / 2 = centered
        const paddleY = 295;         // near the bottom
        const paddleColor = "#2563eb"; // darker blue

        // === DRAW THE BALL ===
        ctx.beginPath();
        ctx.arc(ballX, ballY, ballRadius, 0, Math.PI * 2);
        ctx.fillStyle = ballColor;
        ctx.fill();
        ctx.closePath();

        // === DRAW THE PADDLE ===
        ctx.fillStyle = paddleColor;
        ctx.fillRect(paddleX, paddleY, paddleWidth, paddleHeight);

        // === DRAW A DECORATIVE BORDER (optional) ===
        ctx.strokeStyle = "#7c3aed";
        ctx.lineWidth = 2;
        ctx.strokeRect(5, 5, 470, 310);
    </script>
</body>
</html>

13 Key Takeaways / 重要なポイント

1
JavaScript is a programming language that adds behavior to web pages. It can do math, make decisions, respond to users, and draw graphics.
JavaScriptはWebページに動作を追加するプログラミング言語です。計算、判断、ユーザーへの反応、グラフィックスの描画ができます。
2
The <script> tag adds JavaScript to your page. Place it just before </body>.
<script>タグでJavaScriptをページに追加します。</body>の直前に配置しましょう。
3
Variables store values: let for values that change, const for values that stay fixed. Always use meaningful names.
変数は値を保存します：変わる値にはlet、固定の値にはconst。常に意味のある名前を使いましょう。
4
Numbers and strings are basic data types. Numbers are for math and positions; strings are for text and colors.
数値と文字列は基本的なデータ型です。数値は計算と位置に、文字列はテキストと色に使います。
5
The <canvas> element is a drawing surface. Get its context with getContext("2d") to access drawing methods.
<canvas>要素は描画面です。getContext("2d")で描画メソッドにアクセスします。
6
Canvas coordinates: (0,0) is the top-left. x goes right, y goes DOWN.
Canvas座標：(0,0)は左上。xは右へ、yは下へ増えます。
7
Rectangles use fillRect(); circles use beginPath() + arc() + fill() + closePath().
長方形はfillRect()を使い、円はbeginPath() + arc() + fill() + closePath()を使います。
8
Set fillStyle before drawing. The color applies to everything drawn after it until changed.
描画の前にfillStyleを設定しましょう。色は変更されるまで、その後に描画されるすべてに適用されます。
