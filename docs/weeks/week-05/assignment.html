1. Assignment Overview / 課題概要

In this assignment, you will bring your static ball to life! Using the game loop pattern you learned in Session 5, you will make a ball move smoothly across the canvas.
この課題では、静止しているボールに命を吹き込みます！セッション5で学んだゲームループパターンを使って、ボールをキャンバス上で滑らかに動かします。

This builds directly on your work from Assignment 3 (drawing shapes on canvas). Now instead of a static drawing, you will create a real animation -- the first step toward building the Breakout game!
これは課題3（キャンバスに図形を描く）の直接的な発展です。静止した絵ではなく、本物のアニメーションを作ります -- ブロック崩しゲームを作るための最初のステップです！

Goal: Make a ball move smoothly across the canvas using a draw() function called repeatedly by setInterval().

目標： setInterval()で繰り返し呼び出されるdraw()関数を使って、ボールをキャンバス上で滑らかに動かします。

2. What You Need To Do / やるべきこと

Follow these steps to create your ball animation:
以下のステップに従ってボールアニメーションを作成してください：

Step 1: Set Up the HTML / HTMLを設定する

Create an HTML file with a <canvas> element (480 x 320 pixels) and a <script> tag for your JavaScript code.
canvas要素（480 x 320ピクセル）とscriptタグを持つHTMLファイルを作成します。

Step 2: Get the Canvas Context / キャンバスコンテキストを取得する

Use document.getElementById() to get the canvas, then getContext("2d") to get the drawing context.
document.getElementById()でキャンバスを取得し、getContext("2d")で描画コンテキストを取得します。

Step 3: Create Position Variables / 位置変数を作成する

Define variables for the ball's x and y position. Start the ball somewhere on the canvas (e.g., the center).
ボールのx座標とy座標の変数を定義します。キャンバス上のどこか（例：中心）からスタートします。

Step 4: Create Velocity Variables (dx and dy) / 速度変数を作成する

Define dx and dy variables that control how many pixels the ball moves each frame.
各フレームでボールが何ピクセル動くかを制御するdxとdy変数を定義します。

Step 5: Define a draw() Function / draw()関数を定義する

Create a function called draw that does three things in this order:
以下の3つのことをこの順番で行うdraw関数を作成します：

  1. Clear the canvas with clearRect() / clearRect()でキャンバスをクリア
  2. Draw the ball at the current x, y position using arc() / arc()を使って現在のx, y位置にボールを描画
  3. Update the position by adding dx to x, and dy to y / xにdxを、yにdyを加えて位置を更新

Step 6: Start the Animation with setInterval / setIntervalでアニメーション開始

Use setInterval(draw, 10) to call your draw function every 10 milliseconds, creating smooth animation.
setInterval(draw, 10)を使って10ミリ秒ごとにdraw関数を呼び出し、滑らかなアニメーションを作ります。

Step 7: Add Comments / コメントを追加する

Add comments to your code explaining what each section does. Use // for single-line comments.
各セクションが何をするかを説明するコメントをコードに追加します。単一行コメントには//を使います。

8. Tips for Success / 成功のためのヒント

  • Build your code step by step. First draw a static ball, then add the function, then add setInterval, then add clearRect, then add dx/dy.
    コードを段階的に構築しましょう。まず静止したボールを描き、次に関数を追加し、setIntervalを追加し、clearRectを追加し、最後にdx/dyを追加します。
  • Use the browser console (F12) to check for errors. If the ball does not appear or does not move, there might be a JavaScript error.
    ブラウザコンソール（F12）でエラーを確認しましょう。ボールが表示されないか動かない場合、JavaScriptエラーがあるかもしれません。
  • Experiment with different values for dx and dy. Try dx = 1, dx = 3, dx = 5 to see how speed changes. Try negative values to change direction.
    dxとdyの異なる値を試してみましょう。dx = 1、3、5を試して速度がどう変わるか見てください。負の値で方向が変わります。
  • Start the ball at a visible position. If you start at (0, 0), the ball might move off-screen before you see it. Try starting at the center of the canvas.
    ボールを見える位置から始めましょう。(0, 0)から始めると、見える前に画面外に出てしまうかもしれません。キャンバスの中心から始めてみてください。
  • Read the lecture notes carefully if you get stuck. The complete working code is provided there!
    行き詰まったら講義ノートをよく読みましょう。完全な動作コードがそこに記載されています！

9. Common Mistakes to Avoid / よくある間違い

  • Forgetting clearRect() -- Without it, you get ghost trails instead of smooth animation. The ball leaves a streak across the canvas.
    clearRect()を忘れる -- これがないと、滑らかなアニメーションではなく残像ができます。ボールがキャンバスに軌跡を残します。
  • Putting parentheses in setInterval: setInterval(draw(), 10) -- This calls draw immediately instead of passing it. Write setInterval(draw, 10) without the ().
    setIntervalで括弧をつける：setInterval(draw(), 10) -- これはdrawを渡すのではなくすぐに呼び出してしまいます。()なしでsetInterval(draw, 10)と書きましょう。
  • Not updating the position -- If you forget x = x + dx and y = y + dy inside the draw function, the ball will be drawn in the same spot every frame (no movement).
    位置を更新しない -- draw関数内でx = x + dxとy = y + dyを忘れると、ボールは毎フレーム同じ場所に描かれます（動かない）。
  • Placing setInterval inside the draw function -- This creates a new interval every frame, causing the animation to speed up uncontrollably. Put setInterval outside and after the draw function definition.
    setIntervalをdraw関数の中に書く -- これは毎フレーム新しいインターバルを作り、アニメーションが制御不能に加速します。setIntervalはdraw関数の定義の外側かつ後に書きましょう。
  • Typos in function or variable names -- JavaScript is case-sensitive. mycanvas is NOT the same as myCanvas. Check your spelling carefully!
    関数名や変数名のタイプミス -- JavaScriptは大文字小文字を区別します。mycanvasとmyCanvasは同じではありません。スペルを注意深く確認してください！
