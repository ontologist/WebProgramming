3 What is a Function? / 関数とは？

Before we can animate our ball, we need to understand functions -- one of the most important concepts in all of programming.

ボールをアニメーションさせる前に、関数を理解する必要があります。これはプログラミング全体で最も重要な概念の一つです。

A function is a named, reusable block of code that performs a specific task. Instead of writing the same code over and over, you write it once inside a function and then "call" that function whenever you need it.

関数は特定のタスクを実行する名前付きの再利用可能なコードブロックです。同じコードを何度も書く代わりに、関数の中に一度書いて、必要なときにその関数を「呼び出し」ます。

Analogy / たとえ話: A function is like a recipe. A recipe for making curry has a name ("Make Curry"), a list of ingredients (parameters), and step-by-step instructions (the code inside). You do not rewrite the recipe every time you want curry -- you just follow the same recipe (call the function) again and again.

関数はレシピのようなものです。カレーのレシピには名前（「カレーを作る」）、材料リスト（パラメータ）、手順（中のコード）があります。カレーを作るたびにレシピを書き直す必要はありません -- 同じレシピに従う（関数を呼び出す）だけです。

Why Do Functions Matter? / なぜ関数が重要なのか？

  • Reusability: Write code once, use it many times / 一度書いて何度も使う
  • Organization: Break big problems into small, manageable pieces / 大きな問題を小さく扱いやすい部分に分ける
  • Readability: Give meaningful names to blocks of code / コードブロックに意味のある名前をつける
  • Automation: Functions can be called automatically (like by setInterval) / 関数は自動的に呼び出せる（setIntervalなどで）

4 Defining Functions / 関数の定義

To create a function in JavaScript, use the function keyword followed by a name and curly braces:

JavaScriptで関数を作るには、functionキーワードの後に名前と波括弧を書きます：

function sayHello() {
    console.log("Hello, World!");
}

The Parts of a Function Definition / 関数定義の各部分

Part / 部分 Example / 例 Purpose / 目的
function keyword function Tells JavaScript you are creating a function / 関数を作ることをJavaScriptに伝える
Name sayHello The name you choose to identify it (use camelCase) / 識別用に選ぶ名前（キャメルケース）
Parentheses () Hold parameters (inputs to the function) / パラメータ（関数への入力）を入れる
Curly braces { ... } Contain the code that runs when the function is called / 関数が呼ばれたときに実行されるコード

Important / 重要: Defining a function does NOT run it! It only creates the recipe. The code inside the curly braces will not execute until you call the function.

関数を定義しても実行されません！レシピを作るだけです。波括弧内のコードは、関数を呼び出すまで実行されません。

Naming Rules / 命名規則

  • Use camelCase: drawBall, clearScreen, updatePosition
  • Names should describe what the function does (use verbs): draw, move, calculate
  • Cannot start with a number or contain spaces
  • Cannot use reserved words (like function, var, if)

5 Function Parameters / 関数のパラメータ

Parameters are variables that you pass into a function to make it more flexible. They go inside the parentheses of the function definition.

パラメータは関数に渡す変数で、関数をより柔軟にします。関数定義の括弧の中に入れます。

A Function Without Parameters / パラメータなしの関数

function greetTanaka() {
    console.log("Hello, Tanaka!");
}

greetTanaka(); // Always prints "Hello, Tanaka!"

This function can only greet Tanaka. What if we want to greet different people?

A Function With Parameters / パラメータありの関数

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet("Tanaka");   // prints: Hello, Tanaka!
greet("Suzuki");   // prints: Hello, Suzuki!
greet("Yamada");   // prints: Hello, Yamada!

Now the same function can greet anyone! The parameter name acts as a placeholder that gets filled in when the function is called.

同じ関数で誰でも挨拶できます！パラメータnameは、関数が呼ばれるときに埋められるプレースホルダーとして機能します。

Multiple Parameters / 複数のパラメータ

Functions can accept multiple parameters, separated by commas:

function drawBall(x, y, radius, color) {
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, Math.PI * 2);
    ctx.fillStyle = color;
    ctx.fill();
    ctx.closePath();
}

// Draw different balls at different positions
drawBall(50, 50, 10, "red");
drawBall(150, 80, 20, "blue");
drawBall(250, 120, 15, "#0095DD");

Tip / ヒント: The order of arguments matters! When you call drawBall(50, 50, 10, "red"), the value 50 goes to x, the second 50 goes to y, 10 goes to radius, and "red" goes to color.

引数の順番が重要です！drawBall(50, 50, 10, "red")と呼ぶと、50がxに、次の50がyに、10がradiusに、"red"がcolorに渡されます。

6 From Static to Moving: The Animation Concept / 静止から動きへ：アニメーションの概念

Now that you understand functions, let's use them to create animation. But first, let's understand how animation works on a computer.

関数を理解したところで、アニメーションを作るために使いましょう。しかしまず、コンピュータでアニメーションがどう動くかを理解しましょう。

Analogy / たとえ話: Think of a flipbook (parapara manga / パラパラ漫画). In a flipbook, each page has a picture that is slightly different from the previous page. When you flip through the pages quickly, your eye perceives smooth motion -- even though each page is just a still image.

パラパラ漫画を思い出してください。各ページには前のページとわずかに異なる絵があります。素早くページをめくると、各ページは静止画であるにもかかわらず、目には滑らかな動きに見えます。

Computer animation works exactly the same way:

  • Draw a picture on the screen (one frame)
  • Erase the picture
  • Draw a slightly different picture (the ball has moved a tiny bit)
  • Erase it again
  • Repeat this many times per second

If we repeat this process fast enough (60 times per second or more), the human eye sees smooth, continuous motion -- just like a movie or a video game.

このプロセスを十分に速く（1秒間に60回以上）繰り返すと、人間の目には映画やビデオゲームのように滑らかで連続的な動きに見えます。

Key Insight / 重要なポイント: Animation is not "moving" an object. It is erasing and redrawing the object in a new position, over and over again, very quickly.

アニメーションはオブジェクトを「動かす」ことではありません。オブジェクトを新しい位置に消して再描画することを、非常に速く繰り返すことです。

7 setInterval -- Repeating a Function / setInterval -- 関数を繰り返す

To create animation, we need a way to call our drawing function over and over automatically. JavaScript provides setInterval() for this purpose.

アニメーションを作るには、描画関数を自動的に何度も繰り返し呼び出す方法が必要です。JavaScriptはこの目的のためにsetInterval()を提供しています。

Syntax / 構文

setInterval(functionName, milliseconds);
Parameter / パラメータ Meaning / 意味
functionName The function to call repeatedly (no parentheses!) / 繰り返し呼び出す関数（括弧なし！）
milliseconds Time between each call in milliseconds (1000ms = 1 second) / 各呼び出し間の時間（ミリ秒単位、1000ms = 1秒）

Example: Counting Every Second / 例：毎秒カウント

var count = 0;

function countUp() {
    count = count + 1;
    console.log(count);
}

// Call countUp every 1000 milliseconds (1 second)
setInterval(countUp, 1000);

This will print 1, 2, 3, 4, 5... in the console, one number every second.

For Animation: Use 10 Milliseconds / アニメーション用：10ミリ秒を使う

setInterval(draw, 10);

This calls the draw function every 10 milliseconds, which gives us approximately 100 frames per second -- more than enough for smooth animation.

これはdraw関数を10ミリ秒ごとに呼び出し、1秒間に約100フレームとなります -- 滑らかなアニメーションには十分すぎるほどです。

Common Mistake / よくある間違い: Do NOT put parentheses after the function name in setInterval!

setInterval(draw, 10); -- CORRECT (passes the function)

setInterval(draw(), 10); -- WRONG (calls the function immediately, passes its result)

setIntervalの中で関数名の後に括弧をつけないでください！括弧をつけると、関数を渡すのではなく、すぐに実行してその結果を渡してしまいます。

8 The Problem: Ghost Trails / 問題：残像（ゴーストトレイル）

Let's try a first attempt at animation. We will change the ball's position each frame and draw it:

アニメーションの最初の試みをしてみましょう。各フレームでボールの位置を変えて描画します：

var canvas = document.getElementById("myCanvas");
var ctx = canvas.getContext("2d");
var x = 0;

function draw() {
    // Draw the ball at the current position
    ctx.beginPath();
    ctx.arc(x, 50, 10, 0, Math.PI * 2);
    ctx.fillStyle = "#0095DD";
    ctx.fill();
    ctx.closePath();

    // Move the ball to the right
    x = x + 2;
}

setInterval(draw, 10);

What happens? Instead of seeing one moving ball, you see a long trail of overlapping circles! The old balls are never erased -- they stay on the canvas forever.

何が起こるか？ 1つの動くボールではなく、重なり合った円の長い軌跡が見えます！古いボールは消されることなく、キャンバス上に永遠に残ります。

Ghost Trail Problem / 残像問題
Frame 1:  O
Frame 2:  O O
Frame 3:  O O O
Frame 4:  O O O O
Frame 5:  O O O O O     (a trail of balls, not one moving ball!)

This happens because the canvas is like a painting -- once you draw something, it stays there permanently until you erase it. We need to erase the canvas before drawing each new frame.

キャンバスは絵画のようなもので、一度何かを描くと消すまで永遠に残ります。新しいフレームを描く前にキャンバスを消す必要があります。

9 clearRect -- Erasing the Canvas / clearRect -- キャンバスの消去

The clearRect() method erases a rectangular area of the canvas, making it transparent again.

clearRect()メソッドはキャンバスの四角形の領域を消去し、再び透明にします。

Syntax / 構文

ctx.clearRect(x, y, width, height);
Parameter / パラメータ Meaning / 意味
x X coordinate of the top-left corner to start clearing / 消去を開始する左上のX座標
y Y coordinate of the top-left corner to start clearing / 消去を開始する左上のY座標
width Width of the area to clear / 消去する領域の幅
height Height of the area to clear / 消去する領域の高さ

Clearing the Entire Canvas / キャンバス全体を消去する

To erase everything on the canvas, clear from the top-left corner (0, 0) to the full width and height:

ctx.clearRect(0, 0, canvas.width, canvas.height);

This gives us a completely blank canvas, ready for the next frame.

これでキャンバスが完全に白紙になり、次のフレームの準備ができます。

Tip / ヒント: Using canvas.width and canvas.height instead of hardcoded numbers means your code works correctly even if the canvas size changes.

ハードコードされた数値の代わりにcanvas.widthとcanvas.heightを使うと、キャンバスのサイズが変わってもコードが正しく動作します。

10 The Game Loop Pattern: Clear, Update, Draw / ゲームループパターン：クリア・更新・描画

Now we can combine everything into the most important pattern in game development: the Game Loop.

すべてを組み合わせて、ゲーム開発で最も重要なパターンであるゲームループを作りましょう。

Every game and animation follows this three-step pattern that repeats continuously:

The Game Loop / ゲームループ
    ┌──────────────────────────────────────────┐
    │                                          │
    │   1. CLEAR   - Erase the entire canvas   │
    │                                          │
    │   2. UPDATE  - Change positions/values   │
    │                                          │
    │   3. DRAW    - Draw everything fresh     │
    │                                          │
    └───────────────────┬──────────────────────┘
                        │
                        │  repeat every 10ms
                        │
                        ▼
                  (back to step 1)

Step 1: CLEAR / ステップ1：クリア

Erase everything on the canvas so we have a clean slate.

ctx.clearRect(0, 0, canvas.width, canvas.height);

Step 2: UPDATE / ステップ2：更新

Change the position of objects (move the ball, update scores, etc.).

x = x + dx;
y = y + dy;

Step 3: DRAW / ステップ3：描画

Draw all objects in their new positions.

ctx.beginPath();
ctx.arc(x, y, 10, 0, Math.PI * 2);
ctx.fillStyle = "#0095DD";
ctx.fill();
ctx.closePath();

Why this order? / なぜこの順番？ We clear first so the old frame disappears. We update positions before drawing so the ball appears in its new position. If we drew first and then updated, the ball would always appear one frame behind.

最初にクリアして古いフレームを消します。描画前に位置を更新してボールが新しい位置に表示されるようにします。先に描画してから更新すると、ボールは常に1フレーム遅れて表示されます。

11 Velocity Variables: dx and dy / 速度変数：dxとdy

To control how the ball moves, we use two variables called dx and dy:

ボールの動きを制御するために、dxとdyという2つの変数を使います：

  • dx (delta x) = the number of pixels the ball moves horizontally each frame / 各フレームでボールが水平方向に移動するピクセル数
  • dy (delta y) = the number of pixels the ball moves vertically each frame / 各フレームでボールが垂直方向に移動するピクセル数
var dx = 2;   // move 2 pixels to the right each frame
var dy = -2;  // move 2 pixels upward each frame

Understanding Direction / 方向の理解

Variable / 変数 Positive (+) / 正 Negative (-) / 負
dx Ball moves RIGHT / ボールが右に移動 Ball moves LEFT / ボールが左に移動
dy Ball moves DOWN / ボールが下に移動 Ball moves UP / ボールが上に移動

Remember / 思い出してください: In canvas coordinates, Y increases downward (not upward like in math class). So dy = -2 moves the ball UP, and dy = 2 moves it DOWN.

キャンバス座標では、Yは（数学の授業とは違い）下方向に増加します。なのでdy = -2はボールを上に、dy = 2は下に動かします。

How Movement Works Each Frame / 各フレームでの動きの仕組み

// Each frame:
x = x + dx;  // new x position = old x + horizontal speed
y = y + dy;  // new y position = old y + vertical speed

If x = 100 and dx = 2, then after one frame x = 102. After two frames x = 104. The ball moves 2 pixels to the right every frame!

x = 100でdx = 2の場合、1フレーム後にx = 102。2フレーム後にx = 104。ボールは毎フレーム2ピクセル右に動きます！

Controlling Speed / 速度の制御

The magnitude (size) of dx and dy controls the speed:

  • dx = 1 -- slow movement
  • dx = 2 -- normal movement
  • dx = 5 -- fast movement
  • dx = 10 -- very fast movement

12 Building the Moving Ball Step by Step / 動くボールの段階的構築

Let's build the complete animation from scratch, one step at a time.

完全なアニメーションをゼロから一歩ずつ作っていきましょう。

Step 1: HTML Setup / ステップ1：HTMLの設定

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Ball Animation</title>
</head>
<body>
    <canvas id="myCanvas" width="480" height="320"></canvas>
    <script>
        // JavaScript goes here
    </script>
</body>
</html>

Step 2: Get the Canvas and Context / ステップ2：キャンバスとコンテキストの取得

var canvas = document.getElementById("myCanvas");
var ctx = canvas.getContext("2d");

Step 3: Define Position and Velocity Variables / ステップ3：位置と速度の変数を定義

var x = canvas.width / 2;   // start in the middle horizontally
var y = canvas.height - 30; // start near the bottom
var dx = 2;                 // horizontal speed
var dy = -2;                // vertical speed (negative = upward)

Step 4: Create the draw() Function / ステップ4：draw()関数を作成

function draw() {
    // 1. CLEAR - erase everything
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // 2. DRAW - draw the ball at current position
    ctx.beginPath();
    ctx.arc(x, y, 10, 0, Math.PI * 2);
    ctx.fillStyle = "#0095DD";
    ctx.fill();
    ctx.closePath();

    // 3. UPDATE - move the ball for next frame
    x = x + dx;
    y = y + dy;
}

Step 5: Start the Animation / ステップ5：アニメーションを開始

setInterval(draw, 10);

The Complete Code / 完成コード

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Ball Animation</title>
</head>
<body>
    <canvas id="myCanvas" width="480" height="320"
            style="border: 1px solid #000;"></canvas>
    <script>
        var canvas = document.getElementById("myCanvas");
        var ctx = canvas.getContext("2d");

        // Position variables
        var x = canvas.width / 2;
        var y = canvas.height - 30;

        // Velocity variables
        var dx = 2;
        var dy = -2;

        function draw() {
            // Clear the entire canvas
            ctx.clearRect(0, 0, canvas.width, canvas.height);

            // Draw the ball
            ctx.beginPath();
            ctx.arc(x, y, 10, 0, Math.PI * 2);
            ctx.fillStyle = "#0095DD";
            ctx.fill();
            ctx.closePath();

            // Update position for next frame
            x = x + dx;
            y = y + dy;
        }

        // Start the animation: call draw() every 10 milliseconds
        setInterval(draw, 10);
    </script>
</body>
</html>

Try it! / 試してみよう！ Copy this code into a new HTML file and open it in your browser. You should see a blue ball moving diagonally from the bottom-center toward the upper-right corner of the canvas.

このコードを新しいHTMLファイルにコピーしてブラウザで開いてみてください。青いボールがキャンバスの下中央から右上の角に向かって斜めに動くのが見えるはずです。

What happens when the ball reaches the edge? / ボールが端に達したらどうなる？ Right now, the ball just flies off the screen and keeps going forever! In the next session, we will learn how to detect when the ball hits a wall and make it bounce back.

今のところ、ボールは画面の外に飛び出して永遠に進み続けます！次のセッションでは、ボールが壁に当たったことを検知して跳ね返す方法を学びます。

13 Key Takeaways / 重要なポイント

1
A function is a reusable block of code. Define it with function name() { } and call it with name().
関数は再利用可能なコードブロックです。function name() { }で定義し、name()で呼び出します。
2
Parameters make functions flexible. They are variables that receive values when the function is called.
パラメータは関数を柔軟にします。関数が呼ばれたときに値を受け取る変数です。
3
setInterval(func, ms) calls a function repeatedly. Use 10ms for smooth animation.
setInterval(func, ms)は関数を繰り返し呼び出します。滑らかなアニメーションには10msを使います。
4
clearRect() erases the canvas. Without it, you get ghost trails instead of animation.
clearRect()はキャンバスを消去します。これがないと、アニメーションではなく残像ができます。
5
The Game Loop pattern: Clear, Update, Draw. Every game follows this pattern, repeated many times per second.
ゲームループパターン：クリア・更新・描画。すべてのゲームがこのパターンに従い、1秒間に何度も繰り返します。
6
dx and dy control speed and direction. Positive dx = right, negative dx = left. Positive dy = down, negative dy = up.
dxとdyが速度と方向を制御します。正のdx = 右、負のdx = 左。正のdy = 下、負のdy = 上。
7
Animation = erasing and redrawing very quickly. Like a flipbook, each frame is a still image shown for a brief moment.
アニメーション = 非常に速く消して再描画すること。パラパラ漫画のように、各フレームは一瞬だけ表示される静止画です。
