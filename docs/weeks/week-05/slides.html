The Problem: Our Ball Does Not Move / 問題：ボールが動かない

Right now, our code draws a ball once and then stops. The ball just sits there on the canvas. It does not move.

But in a game, things need to move! How do we make the ball change position over time?

Key Question: How do we go from a static picture to a moving animation?

Answer: We need to draw the ball, erase it, move it slightly, and draw it again -- over and over, very fast!

問題：ボールが動かない

現在のコードはボールを一度描いて終わります。ボールはキャンバスの上に静止しています。

ゲームでは物が動く必要があります！ どうすればボールの位置を時間とともに変えられるでしょうか？

答え： ボールを描いて、消して、少し動かして、また描く -- これを非常に速く繰り返します！

Animation = Flipbook / アニメーション = パラパラ漫画

Think of a flipbook (parapara manga). Each page has a slightly different drawing. When you flip through the pages quickly, the pictures appear to move!

Computer animation works the same way:

  1. Draw a picture (frame 1)
  2. Erase it
  3. Draw a slightly different picture (frame 2)
  4. Erase it
  5. Repeat 60 times per second!

60 frames per second (fps) = the standard speed for smooth animation. Each frame lasts about 16 milliseconds.

アニメーション = パラパラ漫画

パラパラ漫画を思い出してください。各ページには少し違う絵があります。素早くめくると、絵が動いて見えます！

コンピュータアニメーションも同じ仕組みです。1秒間に60回（60fps）絵を描き直すと、滑らかに動いて見えます。

What is a Function? / 関数とは？

A function is a reusable block of code that performs a specific task. Think of it like a recipe:

  • A recipe has a name (e.g., "Make Curry")
  • A recipe has steps (instructions inside)
  • You can follow the recipe (call it) whenever you want
  • You can follow it many times without rewriting the steps

Why functions? Without functions, we would have to copy-paste the same code over and over. Functions let us write code once and reuse it!

関数とは？

関数は特定のタスクを実行する再利用可能なコードのブロックです。レシピのようなものです：

  • レシピには名前がある（例：「カレーを作る」）
  • レシピには手順がある（中の指示）
  • いつでもレシピに従う（呼び出す）ことができる
  • 手順を書き直さずに何度でも従える

Defining a Function / 関数の定義

To create a function, use the function keyword:

function sayHello() { console.log("Hello, World!"); }

The Parts:

  • function -- the keyword that starts the definition
  • sayHello -- the name you choose (use camelCase)
  • () -- parentheses (for parameters, more on this later)
  • { } -- curly braces contain the code that runs

Important: Defining a function does NOT run it! It just creates the recipe. You must call it to run it.

関数の定義

functionキーワードで関数を作成します。

  • function -- 定義を開始するキーワード
  • sayHello -- 自分で選ぶ名前（キャメルケース）
  • () -- 括弧（パラメータ用）
  • { } -- 波括弧に実行するコードを入れる

重要： 関数を定義しても実行されません！呼び出さなければ動きません。

Calling a Function / 関数の呼び出し

To run a function, write its name followed by parentheses:

// Define the function (create the recipe) function sayHello() { console.log("Hello, World!"); } // Call the function (follow the recipe) sayHello(); sayHello(); sayHello();

The code above prints "Hello, World!" three times. We wrote the code once but used it three times!

関数の呼び出し

関数を実行するには、名前の後に括弧を書きます：

上のコードは「Hello, World!」を3回表示します。コードを1回書いて、3回使いました！

Function Parameters / 関数のパラメータ

Parameters let you pass values into a function, making it more flexible:

function greet(name) { console.log("Hello, " + name + "!"); } greet("Tanaka"); // prints: Hello, Tanaka! greet("Suzuki"); // prints: Hello, Suzuki!

Multiple Parameters:

function drawCircle(x, y, radius) { ctx.beginPath(); ctx.arc(x, y, radius, 0, Math.PI * 2); ctx.fill(); ctx.closePath(); } drawCircle(100, 50, 10); drawCircle(200, 80, 20);

パラメータ

パラメータを使うと、関数に値を渡すことができ、より柔軟になります。

複数のパラメータをカンマで区切って渡すことができます。

setInterval -- Repeating a Function / 関数の繰り返し

setInterval() calls a function repeatedly at a fixed time interval:

setInterval(functionName, milliseconds);

Example:

function sayHi() { console.log("Hi!"); } // Call sayHi every 1000 milliseconds (= 1 second) setInterval(sayHi, 1000);

For smooth animation: Use 10 milliseconds as the interval. This gives us approximately 100 frames per second -- more than enough for smooth motion.

setInterval(draw, 10);

setInterval -- 関数を繰り返し呼び出す

setInterval()は関数を一定の時間間隔で繰り返し呼び出します。

滑らかなアニメーションには： 10ミリ秒の間隔を使います。

The Problem: Ghost Trails / 問題：残像

If we just draw the ball in a new position every frame without erasing the old one, we get "ghost trails" -- all the previous positions stay visible!

var x = 0; function draw() { // This draws a new ball but the old one is still there! ctx.beginPath(); ctx.arc(x, 50, 10, 0, Math.PI * 2); ctx.fillStyle = "#0095DD"; ctx.fill(); ctx.closePath(); x = x + 2; } setInterval(draw, 10);

Result: You see a long line of overlapping circles instead of one moving ball!

Solution: Before drawing each frame, we need to erase the entire canvas!

問題：残像（ゴーストトレイル）

古いボールを消さずに毎フレーム新しい位置にボールを描くと、「残像」ができます。以前の位置がすべて見えたままになります！

解決策： 各フレームを描く前に、キャンバス全体を消す必要があります！

clearRect -- Erasing the Canvas / キャンバスを消す

clearRect(x, y, width, height) erases a rectangular area of the canvas. To erase the entire canvas:

ctx.clearRect(0, 0, canvas.width, canvas.height);

What the parameters mean:

  • 0, 0 -- start from the top-left corner
  • canvas.width -- erase all the way to the right edge
  • canvas.height -- erase all the way to the bottom edge

This clears everything on the canvas, giving us a blank slate for the next frame.

clearRect -- キャンバスを消す

clearRect(x, y, width, height)はキャンバスの四角形の領域を消します。キャンバス全体を消すには上のコードを使います。

  • 0, 0 -- 左上の角から開始
  • canvas.width -- 右端まで消す
  • canvas.height -- 下端まで消す

The Game Loop: Clear, Update, Draw / ゲームループ

Every game and animation follows the same pattern, called the Game Loop:

1. CLEAR -- Erase everything

2. UPDATE -- Change positions/values

3. DRAW -- Draw everything in new positions

Repeat forever!

function draw() { // 1. CLEAR ctx.clearRect(0, 0, canvas.width, canvas.height); // 2. UPDATE x = x + dx; y = y + dy; // 3. DRAW ctx.beginPath(); ctx.arc(x, y, 10, 0, Math.PI * 2); ctx.fillStyle = "#0095DD"; ctx.fill(); ctx.closePath(); }

ゲームループパターン：クリア・更新・描画

すべてのゲームとアニメーションはゲームループと呼ばれる同じパターンに従います：

  1. クリア -- すべてを消す
  2. 更新 -- 位置や値を変更する
  3. 描画 -- 新しい位置にすべてを描く

これを永遠に繰り返します！

Velocity: dx and dy / 速度変数

dx and dy are velocity variables that control how fast and in which direction the ball moves:

  • dx = how many pixels to move horizontally each frame
  • dy = how many pixels to move vertically each frame
var dx = 2; // move 2 pixels to the right each frame var dy = -2; // move 2 pixels upward each frame

How direction works:

  • dx positive = move right / dx negative = move left
  • dy positive = move down / dy negative = move up

速度変数：dxとdy

dxとdyはボールの移動速度と方向を制御する速度変数です：

  • dx = 各フレームで水平方向に移動するピクセル数
  • dy = 各フレームで垂直方向に移動するピクセル数

正の値は右/下へ、負の値は左/上へ移動します。

The Complete Moving Ball / 完成：動くボール

Here is the complete code that makes a ball move across the canvas:

var canvas = document.getElementById("myCanvas"); var ctx = canvas.getContext("2d"); var x = canvas.width / 2; var y = canvas.height - 30; var dx = 2; var dy = -2; function draw() { ctx.clearRect(0, 0, canvas.width, canvas.height); ctx.beginPath(); ctx.arc(x, y, 10, 0, Math.PI * 2); ctx.fillStyle = "#0095DD"; ctx.fill(); ctx.closePath(); x = x + dx; y = y + dy; } setInterval(draw, 10);

完成コード

上のコードでボールがキャンバス上を動きます。クリア→描画→更新の順で、10ミリ秒ごとに繰り返します。
